<p>in <a href="http://timheuer.com/blog/archive/2007/08/29/convert-rss-to-json-using-asp-net-javascript-ajax.aspx">my previous post i talked about converting rss data to json</a> using the asp.net ajax javascriptserializer class.  i wanted to use this because i wanted an easier way of interacting with some simple data for a sample i was messing with.  i didn't really see the need to wrap it in a web service call, etc. and in the end i wanted json data.  period.</p>
<p>so why did i want the json data?  to mess with it in <a rel="tag" href="http://silverlight.net">silverlight</a> of course! what i wanted to do was display my rss feed data in a different way, just out of curiosity.  my goal: display my rss data, in my own handwriting, on a post it note.  in silverlight.  here's what i did.</p>
<p><strong><u>the font</u></strong></p>
<p>first, i needed my own handwriting in a true type font (fyi: silverlight supports using open type and true type fonts).  so i went on over to the <a href="http://www.microsoft.com/windowsxp/downloads/powertoys/tabletpc.mspx">tablet pc powertoys</a> and grabbed the "My Font Tool" -- this is a tool that allows you to ink the alphabet and then it turns it into a true type font for you.  i did that process and boom, i bring you "Tim Heuer Normal":</p>
<p><img alt="Tim Heuer Normal TrueType" src="http://s3.amazonaws.com/timheuer-img/timheuer-ttf.png" /></p>
<p>okay, great, that step is done.  on to the post it.  headed on over to <a href="http://www.istockphoto.com/">istockphoto.com</a> for a quick gander at some good stuff and snagged a good one for me.  downloaded it with my credits, used <a href="http://www.microsoft.com/expression">expression design</a> to clean it up and it was ready to go, png-style.</p>
<p><strong><u>the silverlight application</u></strong></p>
<p>now on to silverlight...i started using expression blend to create my project.  a simple canvas that had my post-it note image on it and then added some TextBlock elements to represent the rss item title and text:</p>
<p><img alt="" src="http://s3.amazonaws.com/timheuer-img/postit-phase1.png" /></p>
<p><strong><u>the hosting page for the silverlight control</u></strong></p>
<p>now in the page that will be hosting the control i had to add the reference to my JSON-ized rss data using my asp.net handler that i previously wrote about.  i ran into a snag here as i suddenly realized that my json-izer just emitted json data.  that's it.  it wasn't referenced in and javascript variable or anything.  in some situations maybe that is fine, but i needed to "set" a variable to the json data to be able to reference the object later.  i quickly added a param to my json-izer to specify a callback method.  what this would do (if present) would emit javascript that basically called a function.  so if you pass in "c=evalRss" it would result in emitting:</p>
<pre class="csharpcode">evalRss([{...json data...}]);</pre>
<p>this would allow me to set the object data to a variable.  in my hosting page i can now add a reference to my json-izer as such:</p>
<pre class="csharpcode"><span class="kwrd">&lt;</span><span class="html">script</span> <span class="attr">src</span><span class="kwrd">="http://blah/rss2json.ashx?u=http://feeds.feedburner.com/timheuer&amp;c=evalRss"</span><span class="kwrd">&gt;&lt;/</span><span class="html">script</span><span class="kwrd">&gt;</span></pre>
<p>of course i needed to implement the function so i added a simple javascript variable and the callback function:</p>
<pre class="csharpcode"><span class="kwrd">var</span> rss = <span class="kwrd">null</span>;<br /><span class="kwrd">function</span> evalRss(objGraph) <br />{ <br />    rss = objGraph; <br />}</pre>
<p>as emiril would say "bam" -- now i have an "rss" object to mess around with...okay, moving on.</p>
<p><strong><u>setting the textblock elements to the rss data</u></strong></p>
<p>in my quick sample here, i'm only going to use the latest item in the rss feed, but read later on how you could use them all...remember, sample here folks.</p>
<p>i already had a default handleLoad function that was stubbed out for me and wired up by expression blend, so i'll go ahead and use that.  in the handleLoad function i take the control reference, find my TextBlock elements and set them to my rss data (remember, since my json-izer was a script reference, the json data is already there):</p>
<pre class="csharpcode">handleLoad: <span class="kwrd">function</span>(control, userContext, rootElement) <br />{ <br />   <span class="kwrd">this</span>.control = control; <br />   <span class="kwrd">this</span>.header = control.content.findName(<span class="str">"ItemHeader"</span>); <br />   <span class="kwrd">this</span>.itemtext = control.content.findName(<span class="str">"ItemContent"</span>); <br />   <span class="kwrd">this</span>.header.Text = rss[0].Channel.Items[0].Title; <br />   <span class="kwrd">this</span>.itemtext.Text = rss[0].Channel.Items[0].Description; <br />}</pre>
<p>okay, whew, got that far.  so when i run the application i get:</p>
<p><img alt="RSS Data" src="http://s3.amazonaws.com/timheuer-img/postit-phase2.png" /></p>
<p>yeah, data.  yes, i see it to.  the html.  blech.  i'll have to think about that.  but for now, let's move on.  now i did notice that my text blabbed on beyond the image height and for this sample i didn't like that.  no problem...back to blend.  i created a rectangle (no stroke, no fill) that was the same left/top/width/height as my ItemContent TextBlock.  then with a ctrl+click on both those elements (rectangle, textblock) i headed up to the object menu, select path...make clipping path.  done.  my textblock now is clipped at the shape of the rectangle (you'll not the Clip data added in the textblock attributes).</p>
<p><strong><u>embedding the font</u></strong></p>
<p>but wait, i'm not done!  i said that i wanted this in my handwriting.  sweet, i'll just set the textblocks to my font...er...um...wait.  argh.  that's not supported (yet).  no worries there is a way!  here's how to go about it (or at least one way).  first add your true type font to your project.</p>
<blockquote>
<p><strong>NOTE:</strong> fonts usually have licenses with them.  you need to make sure you understand your font's license before you start embedding it all over the place.  in my case, i contacted tim heuer headquarters and they said it was okay.</p>
</blockquote>
<p>now, how do i get the font on the textblock.  here's the process i used.  first, i new'd up a downloader object to get silverlight to download the font to the control and thus have access to it.  i put this in my handleLoad event handler because i want the font to be there asap:</p>
<pre class="csharpcode"> <span class="kwrd">this</span>.downloader = control.createObject(<span class="str">"downloader"</span>); <br /> <span class="kwrd">this</span>.downloader.addEventListener(<span class="str">"completed"</span>, <br />        Silverlight.createDelegate(<span class="kwrd">this</span>, <span class="kwrd">this</span>.handleFontDownloaded)); <br /> <span class="kwrd">this</span>.downloader.open(<span class="str">"GET"</span>, <span class="str">"timheuer.ttf"</span>); <br /> <span class="kwrd">this</span>.downloader.send();</pre>
<p>you'll notice that i added an event listener for the "completed" event of the downloader.  that is where the magic happens with two functions: setFontSource and fontFamily (okay, one function, one attribute).  the SetFontSource (setFontSource in javascript) expects a downloader object as the only parameter.  we're going to use the this.downloader object we created above.  we'll do the setFontSource-ing on our TextBlock elements.  after we call that function, then we can go back to the TextBlock elements and set the fontFamily attribute to the <strong>named</strong> font value.  i call out <strong>named</strong> because even though your font <strong>file</strong> might be "timheuer.ttf" that is not the name of the font family.  all this is done in the handleFontDownloaded event handler and looks like this when completed:</p>
<pre class="csharpcode">handleFontDownloaded: <span class="kwrd">function</span>(sender, eventArgs) <br />{ <br />  <span class="kwrd">this</span>.header.setFontSource(sender); <br />  <span class="kwrd">this</span>.itemtext.setFontSource(sender); <br />  <span class="kwrd">this</span>.header.fontFamily = <span class="str">"Tim Heuer Normal"</span>; <br />  <span class="kwrd">this</span>.itemtext.fontFamily = <span class="str">"Tim Heuer Normal"</span>; <br />}</pre>
<p>so what is happening here?  well silverlight has downloaded the font file using the downloader.  we've passed the downloader reference (which is the sender in the downloader completed event) to the setFontSource property of an element (in our case the TextBlock).  once the font source has been set we set the fontFamily.  so how does silverlight know what to look for?  magic i tell you, magic.  because we've set the font source to a downloader (which downloaded a file), that is the location it is now looking for the fontFamily value...and iterates through that downloader until it finds a match...in our case a true type font named "Tim Heuer Normal" to use.  once we have that i re-run the application and here we have:</p>
<p><img alt="RSS Data with Tim Heuer Font" src="http://s3.amazonaws.com/timheuer-img/postit-phase3.png" /> </p>
<p>awesome.  </p>
<p><strong><u>what about more than one font?</u></strong></p>
<p>but what if you wanted multiple fonts?  would you need multiple downloaders?  no.  you see, the downloader supports compressed (zipped) files.  and it doesn't just have to be fonts.  it could be a mixture of media, fonts, xaml assets, etc.  in fact, that is how <a href="http://zerogravity.terralever.com">zero gravity</a> gets all its assets for the game prior to play.  if you had a zip file that you added the "timheuer.ttf" file into, really the only thing that would change would be the downloader.open() function to downloader.open("GET", "myzipfile.zip") -- your setFontSource and fontFamily code wouldn't change!  if i had two fonts in the zip file then i could change the header to something like this.header.fontFamily = "Knockout C" or whatever other true type font is in the zip file.</p>
<p><strong><u>having some fun</u></strong></p>
<p>now that we have all that wired up, we can move things around and make it look like i posted a note just sticking it on the frigde or something -- not linear.  i can simply add a rotatetransform on my canvas and i'll get:</p>
<p><img alt="" src="http://s3.amazonaws.com/timheuer-img/postit-phase4.png" /></p>
<p>the fonts, etc. change no problems.</p>
<p>so my little quest was complete.  i got my handwriting in my rss feed in silverlight.  yippeee.  now if only i had an investor.</p>
<p><strong><u>the issues and controls</u></strong></p>
<p>there were issues as you saw.  first the characters.  the "my font" tool doesn't ask me to write every possible characters, so if i used some that weren't in my font, blech.  the point here is that you'd likely use a cool font that was professional.</p>
<p>the html -- yes.  i'll have to think about this one as i do have a use for this concept coming up.  my rss feed is html.  silverlight doesn't really have an "html" survace...it isn't interpreting &lt;p&gt; as a paragraph break, etc.  in this sample, it might be problematic, but other areas where you have text in textblocks and are embedding fonts you shouldn't have that issue.</p>
<p>what if you wanted to display all rss feeds?  would you need 10 textblocks and set them 10 times?  well, kinda.  my suggestion here is to use the control approach.  i already have the xaml for my post-it display.  i could take that, put it in a control, stuff it into the zip file <strong>with my font</strong> and have a control i could reference.  (for an example screencast on user controls in v1.0 <a href="http://timheuer.com/blog/archive/2007/08/19/implement-user-control-in-silverlight-user-controls.aspx">see this link</a>) for example, i could encapsulate the post-it in a control that looked something like this:</p>
<pre class="csharpcode">RssNotebook.RssItem = <span class="kwrd">function</span>(control, target, header, desc, x, y, downloader) <br />{ <br />  <span class="kwrd">this</span>.header = target.findName(<span class="str">"ItemHeader"</span>); <br />  <span class="kwrd">this</span>.itemtext = target.findName(<span class="str">"ItemContent"</span>); <br /><br /><br />  <span class="kwrd">this</span>.header.setFontSource(downloader); <br />  <span class="kwrd">this</span>.itemtext.setFontSource(downloader); <br />  <span class="kwrd">this</span>.header.fontFamily = <span class="str">"Tim Heuer Normal"</span>; <br />  <span class="kwrd">this</span>.itemtext.fontFamily = <span class="str">"Tim Heuer Normal"</span>; <br />  <span class="kwrd">this</span>.header.Text = header; <br />  <span class="kwrd">this</span>.itemtext.Text = desc; <br />  target[<span class="str">"Canvas.Top"</span>] = y; <br />  target[<span class="str">"Canvas.Left"</span>] = x; <br />}</pre>
<p>then i can extract the RssItem xaml, put that in a file and add it to a zip with my timheuer.ttf file.  I now have a zip with RssItem.xaml and timheuer.ttf.  now with that i can alter my downloader completed event to look like this:</p>
<pre class="csharpcode">handleFontDownloaded: <span class="kwrd">function</span>(sender, eventArgs) <br />{ <br /><span class="kwrd">  var</span> top = 0; <br />  <span class="kwrd">for</span> (<span class="kwrd">var</span> i=1; i&lt;3; i++) <br />  { <br />    <span class="kwrd">var</span> postit = <span class="kwrd">this</span>.control.content.createFromXaml( sender.getResponseText(<span class="str">"RssItem.xaml"</span>), <br />        <span class="kwrd">true</span>); <br />    <span class="kwrd">this</span>.root.children.add(postit); <br />    <span class="kwrd">new</span> RssNotebook.RssItem(<span class="kwrd">this</span>.control, postit, rss[0].Channel.Items[i].Title, <br />                 rss[0].Channel.Items[i].Description, 0, top, sender); top += 100; <br />  } <br />}</pre>
<p>so what you see happening here is two things.  first let me be clear that this is still a sample, and for the control part i added a scale transform to reduce the size of the image just so i could display it here.  okay, what is happening is that i'm iterating through two of the rss items and createing the RssItem control within that iteration (createFromXaml) getting the xaml out of the downloaded zip file.  then in that control i'm doing the fontSource-ing and fontFamily-ing for the TextBlocks represented in their control.  </p>
