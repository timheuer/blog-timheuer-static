<p>i've been working on minor enhancements/changes to feedreader and 2.1 is tentatively scheduled to include the following:</p>
<ul>
<li>pre-authenticating user request to allow user authenticated proxy servers</li>
<li>moving the proxy server and cache settings to shared web part properties only</li>
<ul>
<li>this is so that each user would not be required to change (or be able to) this information for more granular control over performance</li></ul>
<li>changing the feed url listing in the toolpart to use newlines instead of separating each url by a “;“ character (note: some url's have a semicolon as a legitimate character)</li>
<li>adding the ability for the description to only display N number of lines</li>
<li>enabline expand/collapse ability on the feed item nodes, so that if description is displayed it will display collapsed and the user can view it but not consume massive amounts of real estate</li></ul>
<p>so i am looking for other suggestions (if any).</p>
<p>one problem i have that is not going to make the upgrade fun for anyone is whether or not i should change the versioning.  technicall i should as it is a different version...however <a title="SharePoint" href="http://www.microsoft.com/sharepoint" target="_blank">sharepoint</a> doesn't make it easy to <a href="http://msdn.microsoft.com/library/default.asp?url=/library/en-us/dnspts/html/sharepoint_UpgradingaWebPartAssembly.asp">upgrade web part assemblies </a>without using assembly redirection (which would require modification to the <a title="SharePoint" href="http://www.microsoft.com/sharepoint" target="_blank">sharepoint</a> web.config)...any ideas on the upgrade are appreciated...the goal being minimal (preferrably none) changes other than a new deployment.</p>
