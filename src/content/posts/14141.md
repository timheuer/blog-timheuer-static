<p>in a few weeks microsoft will be releasing the release candidate for <a href="http://silverlight.net" rel="tag">silverlight</a> 1.0.  on joe stegman's blog, he <a href="http://blogs.msdn.com/jstegman/archive/2007/06/06/more-v-1-0-changes.aspx">posted about breaking changes</a>, etc.  moving forward once the RC is released, the api will be stable and should have no further breaking changes between RC and final release.  however, there will be breaking changes in the RC from beta.  if you have a silverlight application, now is the time to start preparing for what you may have to change.</p> <p>microsoft has prepared a preview SDK release containing:</p> <ul> <ul> <li>new silverlight.js detection/installation file for both beta and RC</li> <li>breaking changes document highlighting differences between beta and RC</li> <li>updated Visual Studio template with control embedding template</li> <li>a EULA that governs the legal use of silverlight.js, etc. (note: the updated license makes it clear that silverlight.js code is not allowed to be modified)</li></ul></ul> <p><a href="http://folders.live.com/self.aspx/kocj18oD+kQ/Silverlight/Silverlight%201.0%20SDK%20RC%20Preview.zip">download this preview SDK here</a>.  using this you should be able to get a head start on the changes.  this SDK does NOT contain the RC of 1.0 itself, rather guidance for when it is released...but the breaking changes document, etc. should be extremely helpful for your preparation.</p> <div class="wlWriterSmartContent" id="scid:0767317B-992E-4b12-91E0-4F059A8CECA8:fb87f95f-d0a6-4238-9e1f-ae92e73647f5" style="padding-right: 0px; display: inline; padding-left: 0px; padding-bottom: 0px; margin: 0px; padding-top: 0px"></div>
