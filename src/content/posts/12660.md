<p>i&rsquo;m pondering another thought from the same individual who <a href="http://timheuer.com/blog/archive/2006/06/15/12658.aspx">posed a previous question</a> to me.&nbsp; part of the comment was also regarding the release of software.</p>
<p>this came about after news of <a href="http://blogs.msdn.com/jasonz/archive/2006/06/09/624629.aspx">WinFX being officially renamed to .NET Framework 3.0</a>.&nbsp; some of the comments made were dealing with learning a new framework and wishing microsoft would slow down the train on releases.&nbsp; in my response to the comment, i had to think about releases&hellip;here&rsquo;s a comparison (again using PHP as a comparitor because the comment was on an asp.net group)&hellip;</p>
<ul>
<li>Feb 2002: .NET 1.0 released (beta starting in 1999)</li>
<li>May 2003: .NET 1.1 released</li>
<li>Nov 2005: .NET 2.0 released (beta starting in 2003)</li>
<li>EOY 2006: .NET 3.0 to be released (which is really .NET 2.0 +)</li></ul>
<p>compare that with PHP (only the latest two major versions)</p>
<ul>
<li>May 2000: PHP v4 released (beta starting in 1999)</li>
<li>Jul 2004: PHP v5 released (beta starting in 2003)</li></ul>
<p>so when i look at these, i don&rsquo;t see a real difference, but yet there are no complaints over releasing new versions of PHP which do include new features (and even broken back compat features)!&nbsp; I see .NET 1 to .NET 2 (i&rsquo;m not counting the dot release) as a 3 year release cycle.&nbsp; &ldquo;.NET 3.0&rdquo; is another year after that, but no significant ground-breaking core framework features&hellip;again it is framework+.</p>
<p>so i wonder, do we release software too quickly?&nbsp; being on the inside, i consistently see NUMEROUS requests for more frequent releases.&nbsp; in fact, in our interactions with the open source community, that is one of the top 3 criticisms&hellip;not releasing soon enough.</p>
