<p>Today Visual Studio 2008 has released SP1 which not only brings some fixes, but also is an added value service pack, bringing some new functionality to WPF as well as enabling a “client” deployment pack of the .NET framework so that those deploying .NET framework with your client applications can have a much smaller footprint (by about 80+%).</p>
<p><strong>With the release of SP1 for Visual Studio 2008 today, the </strong><a href="http://silverlight.net"><strong>Silverlight</strong></a><strong> team has also updated their tools for Silverlight 2 Beta 2</strong>.  Read again: a tools update for Silverlight 2 Beta 2 is needed and available for you.  If you install Visual Studio 2008 SP1 and your Silverlight projects aren’t working anymore, that’s why!</p>
<p>Here’s the download links:</p>
<ul>
    <ul>
        <li><a href="http://www.microsoft.com/downloads/details.aspx?FamilyID=27673c47-b3b5-4c67-bd99-84e525b5ce61&amp;DisplayLang=en">Visual Studio 2008 SP1 (release and full download)</a> </li>
        <li><a href="http://www.microsoft.com/downloads/details.aspx?FamilyId=50A9EC01-267B-4521-B7D7-C0DBA8866434&amp;displaylang=en">Silverlight 2 Beta 2 tools refresh for SP1</a> </li>
    </ul>
</ul>
<p>Please note that this update to the Silverlight tools can be used for Visual Studio 2008 RTM as well as the just-released SP1, but NOT for VS 2008 SP1 beta.  Yeah, if that’s mind blurring, let’s boil it down:</p>
<p>Do you want to install VS2008 SP1 (release) and still work with Silverlight 2 applications?  If yes, download the updated Silverlight tools and install <strong>after</strong> you install VS208 SP1.</p>
<p>I hope that makes your decision easier.  <strong>Oh and if you installed SP1 Beta at any time, </strong><a href="http://blogs.msdn.com/heaths/archive/2008/08/11/vs-2008-sp1-beta-must-be-removed-prior-to-installing-the-release-of-vs-2008-sp1.aspx"><strong>check out this post from Heath</strong></a>.</p>
