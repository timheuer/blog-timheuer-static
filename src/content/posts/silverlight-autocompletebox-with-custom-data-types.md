<p>The other day I wrote a simple little <a href="http://silverlight.net">Silverlight</a> application using a DataGrid to help navigate the <a href="http://timheuer.com/blog/archive/2008/10/31/teched-north-america-dvd-silverlight-update.aspx">TechEd DVD contents</a>.&#160; My code was admittedly quick and dirty.&#160; I loaded up some data, and based on some events re-filtered and re-bound that data.&#160; After thinking about it I am not sure why I didn’t just use some existing controls to help me do that work.&#160; I found that AutoCompleteBox from the new <a href="http://codeplex.com/Silverlight">Silverlight Toolkit</a> would do this for me.</p>

<p>If you wire up the AutoCompleteBox.ItemsSource to my List&lt;&gt; nothing will happen.&#160; Why?&#160; Well the control doesn’t know how to understand the data exactly.&#160; So we have to help it.&#160; It knows the data is there but doesn’t know what to return based on the filter.</p>

<p>This is simply done using the ItemFilter property on the control.&#160; We can easily provide a Lambda expression to help the control understand the filter.&#160; After setting the ItemsSource property to the results of my LINQ query, I add the expression to help the control understand the data:</p>

<p>That’s it.&#160; I’ve now defined the filter for the control so as the user types data it does the appropriate filtering logic and displays the results.&#160; I’ve chosen to still display the results as a DataGrid for easy reading in this application, but could have also used an ItemTemplate if I desired.&#160; The ItemFilter expects a search predicate to be passed to it.</p>

<p>So the resulting application has the same capabilities, but a lot less code for me to write.&#160; Here’s the ending result:</p>

<p>So if you need to implement AutoCompleteBox and have it search/filter your custom types, be sure to supply your own ItemFilter to help the control know where to look!</p>
