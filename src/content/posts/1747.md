<p>last night i had the opportunity (as i do every month, so i guess it&rsquo;s not an opportunity, but rather a regular thing) to go to the <a href="http://www.phxjug.org/" target="_blank">phoenix java user group </a>meeting.</p>
<p><em>why are you going to that?</em> why not.&nbsp; i&rsquo;m a software developer.&nbsp; they talk about software development.&nbsp; yeah, they talk about java, but it&rsquo;s always interesting to see why people are using the wrong technologies.&nbsp; oops did i say that out loud?&nbsp; i am kidding of course.&nbsp; i go because it is interesting to see what *all* technologies are doing to adapt to the changing needs of business and software development communities.&nbsp; java (sun) is no differnent than .net (microsoft) in that regard.</p>
<p>the first half of the meeting was their president (todd ellermann) talking about his participation in the technology summit held last month.&nbsp; he did an overview for the group of his perspective of the event.&nbsp; another overview can be found <a href="http://timheuer.com/blog/archive/2005/03/25/1726.aspx" target="_blank">here</a> from another attendee&rsquo;s perspective.&nbsp; i will not go into detail here&hellip;it is better, quite frankly, to hear it from a non-ms mouth.</p>
<p>the next part of the meeting was interesting&hellip;very interesting to me.&nbsp; a &ldquo;java ambassador&rdquo; (i interpreted this to be like the ms evangelism team) was there to talk/demo about <a href="http://developers.sun.com/prodtech/javatools/jscreator/index.jsp" target="_blank">java studio creator </a>and <a href="http://java.sun.com/j2ee/javaserverfaces/" target="_blank">java server faces</a>.&nbsp; for the purposes here, i am choosing to draw a direct coorelation to visual studio and asp.net &mdash; because that&rsquo;s exactly what i saw last night.</p>
<p>to preface, the java guy (tom, who did a great job by the way) paved the way by talking about who the target is for these tools/technology (java studio is a tool, server faces is the technology).&nbsp; he was very clear that one of the primary (not the only) targets was the vb developer (you can also see sun&rsquo;s feeling on this as well on their site: <a href="http://blogs.sun.com/roller/page/brewin/20050403#end_of_an_era">http://blogs.sun.com/roller/page/brewin/20050403#end_of_an_era</a>.&nbsp; tom went as far to say &ldquo;we want to conver the vb developer.&rdquo;&nbsp; in describing the tool (java studio) he described it as a fully integrated ide with tools built in for editing, debugging, testing, etc. (<a href="http://www.microsoft.com/vstudio" target="_blank">sound familiar</a>) and specifically called out by saying &ldquo;we wanted to make it look like the ms tools&rdquo; &mdash; now i interpreted that as not copying, but rather in the effort to sway the vb developer, the tools had to look/feel familiar.&nbsp; there was&nbsp;a lot of marketing speek about how cool it is etc.&nbsp; then he gave some dates.&nbsp; java server faces/studio was announced and initially launched in the fall of 2003&hellip;keep that in your short-term memory bank for the remainder of this read.</p>
<p>okay, on to the demo.&nbsp; he launches java studio creator on his java desktop machine (yes, running java desktop on a linux distr.).&nbsp; wow, the ide looks familiar&hellip;it has a design surface, a project explorer, output windows, widget panels, server explorer, properties editing ui, etc., etc.&nbsp; in case you don&rsquo;t get where i&rsquo;m going here&hellip;let me rephrase this: it has a design surface with tabs for visual design and jsp source (design surface for designer and html views), a project explorer (solution explorer), output windows (output windows, immediate, etc.), widget panels (toolbox), server explorer (duh, same thing), properties editing (duh, same thing), etc., etc. &mdash; i mean, it looked like vstudio!&nbsp; now i admit, this is the first time i&rsquo;ve seen this, so granted it may have been around before&hellip;but remember the date (fall 2003)&hellip;i mean, visual studio 2002 was fall 2001.&nbsp; for once i felt inside (yes, i have to admit it) that i felt *hopefully* someone may see this java studio as a follower&hellip;it always seems to be the other way around.</p>
<p>this was more pronounced when the java server faces were being demonstrated and discussed.&nbsp; let me put this in phrases that make it simple.&nbsp; what i saw in java server faces was everything that has existed in asp.net.&nbsp; seriously, everything&hellip;here&rsquo;s some interesting comparisons.</p>
<ul>
<li>validators &mdash; they had required validators, range validation, operator validation, etc.</li>
<li>widgets (controls) &ndash; there was listbox, text field, etc, etc. &mdash; all with data binding capabilites (read more on that later)</li>
<li>&ldquo;backing beans&rdquo; (read: codebehinds) for each JSP page created.</li>
<li>there was an application/session backing bean for the web app &mdash; read more on that later</li>
<li>the ide had &ldquo;code clips&rdquo; (read: code snippets)</li>
<li>the ide had the &ldquo;squiggly&rdquo; lines indicating there was a bad reference or bad code (i&rsquo;ve always said intellisense is the most underrated application)</li></ul>
<p>pretty much asp.net stuff that i&rsquo;ve been used to working with.&nbsp; what was interesting was when i asked (naively of course because i really didn&rsquo;t know) <em>is this stuff available now?</em>&nbsp; the response form the gentleman in front of me was kinda funny in a way.&nbsp; he turned and said with kind of a smile/smirk (they knew i was from ms in the room) <em>yeah, it&rsquo;s been around, i was using it this morning</em>.&nbsp; now i could have easily b-slapped the comment to <em>that&rsquo;s nice, because i was waiting for your technology to catch up to what i&rsquo;ve been doing since 2001</em> but i didn&rsquo;t.</p>
<p>at any rate, it was ironic&hellip;the stuff was the same philosophy, goals, etc. that visual studio.net and asp.net have been doing since the inception.&nbsp; what was the most amazing was everyone in the room saying &ldquo;cool&rdquo; and all the oohs and aahs going on&hellip;i wanted to scream so bad, but didn&rsquo;t&hellip;well okay, later i did.</p>
<p>there were some cool things in the tool that i did like and think ms should adopt&hellip;here&rsquo;s some snippets..er i mean clips ;-):</p>
<ul>
<li>page navigation &mdash; they had a UI tool in the ide to basically create the sitemap xml file that could be used to help auto bind navigation rules &mdash; it would be cool if whidbey has this to create the sitemap.xml file</li>
<li>drag binding &mdash; you could drag/drop your data objects on top of a control (i.e., listbox) and the databinding would be done for you &mdash; it pops up an intuitive &ldquo;what fields&rdquo; kind of thing &mdash; i think this is better in whidbey with the smart tags concept</li>
<li>validation &mdash; for the simple ones, like required, it was a part of the control&hellip;basically you set a control&rsquo;s property to required=&rdquo;true&rdquo;</li>
<li>ui mapping of control values to scope variables (session, application, etc.)</li></ul>
<p>there were also some things i think that need improvement (as well did the audience)&hellip;</p>
<ul>
<li>the session bean was cool everyone thought, but tom couldn&rsquo;t comment if it was tied to http session but did mention that there is no session failover (so clearly it is then), which caused some stomach growls &mdash; hey, asp.net has a state server and other ways for managing that in a clustered environmnt :-D</li>
<li>on the databinding there seemed to be an assumption the source would be JDBC rowsets &mdash; this didn&rsquo;t bode well with the audience &mdash; did i mention whidbey has a provider model and abstracts the databinding objects to a point of &ldquo;who cares&rdquo; what it is on the backend?</li></ul>
<p>at any rate, i&rsquo;m babbling&hellip;my point being this was cool&hellip;it really was&hellip;but there definitely was some satisfaction to me knowing that i&rsquo;ve already been doing this for a while&hellip;a long while.</p>
