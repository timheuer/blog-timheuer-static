<p>please please&#8230;if you provide an installer with program group shortcuts&#8230;make a program group shortcut to the uninstaller!</p>
<p><em>but, windows has an uninstall control panel where you can get to it</em></p>
<p>sure, but you provide program group shortcuts to help files that are also in the products, so the &#8220;you can get to it another way&#8221; isn&#8217;t a valid excuse if you already provide shortcuts to things you can get to in other ways.</p>
<p>also&#8230;if you develop .net components, please strong-name them.  if you claim to be fully CAS compliant, then you must&#8230;otherwise you lie ;-) &#8212; and if you think i&#8217;m lying, you&#8217;re wrong &#8212; try implementing a CAS policy using publey key token without a strong named assembly ;-) &#8212; sure that&#8217;s one implementation and you can do it other ways, but to be fully CAS compliant you should be able to support *any* way the consumer might utilize your component!</p>
