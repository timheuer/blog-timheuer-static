<p>now that the <a href="http://timheuer.com/blog/archive/2007/04/27/14055.aspx">virtual earth version 5 sdk/control is out</a> and available, some that may want to quickly move to v5 may find some refactoring required.  for me, it was pushpins.  when using v4 of vritual earth, i would normally create a VEPushpin first like this:</p> <blockquote><pre class="csharpcode"><span class="kwrd">var</span> pin = <span class="kwrd">new</span> VEPushpin(
        someId, <span class="rem">// unique identifier of the pin </span>
        latLon, <span class="rem">// a VELatLong object</span>
        <span class="str">"images/custompin.png"</span>, <span class="rem">// custom pin object</span>
        <span class="str">"some title"</span>, <span class="rem">// title</span>
        <span class="str">"some details"</span>, <span class="rem">// details</span>
        <span class="str">"pinIcon"</span>, <span class="rem">// icon style</span>
        <span class="str">"pinTitle"</span>, <span class="rem">// pin title style</span>
        <span class="str">"pinDetails"</span>); <span class="rem">// pin details style</span>
    
map.AddPushpin(pin);</pre></blockquote>
<p>when i changed my app to use the v5 control, that instantly broke.  in v5 the pushpin is elevated to the same level as other shapes (or other shapes are downleveled to the same place, whatever you want to think of it).  now when you want to add a pin with some customization, you will want to create a shape like this:</p>
<blockquote><pre class="csharpcode"><span class="kwrd">var</span> shape = <span class="kwrd">new</span> VEShape(
                  VEShapeType.Pushpin, <span class="rem">// the type of shape</span>
                  [latLong] <span class="rem">// a VELatLong object</span>
                 );

shape.SetCustomIcon(pinIcon); <span class="rem">// if you wanted to set a custom icon</span>

shape.SetTitle(<span class="str">"&lt;div class=\"pinTitle\"&gt;"</span> + title + <span class="str">"&lt;/div&gt;"</span>);
shape.SetDescription(<span class="str">"&lt;div class=\"pinDetails\"&gt;"</span> + details + <span class="str">"&lt;/div&gt;"</span>);

map.AddShape(shape);</pre></blockquote>
<p>as you can see basically it isn't too much work, but essentially everything you add is a "shape" (either a pushpin, polyline, polygon).  the only thing i don't like about the new v5 way of doing things is in the v4 way, you could set the css class of the icon, title, details and it would apply it.  i think that was very simplified and allowed me greater control and separation of my styles from my code.  in v5 i now have to do that within my script.  :-( -- hey VE guys, can we change this?</p>
<p>of course, all this is documented in the updated <a href="http://dev.live.com/virtualearth/sdk/" rel="tag">interactive sdk</a>, but i thought it might find it helpful real quick-like for those who may happen to read this :-)</p>
