<p>well, after doing a round of demos on <a href="http://atlas.asp.net/">atlas</a> and having to work with the virtual earth map stuff (argh), i decided that geocoding sucked and i&rsquo;m surprised it isn&rsquo;t a base offering in more apis.&nbsp; i guess i don&rsquo;t understand why, when maps, etc. are being handed out like candy, but a simple method/api/whatever to provide the address and get the lat/long back isn&rsquo;t &mdash; after all, the maps are doing it anyway.</p>
<p>at any rate, i found a few providers, the free one of which seems to be running off of a sixth graders pda&hellip;and can only handle about 1/2 simultaneous request.&nbsp; i started using it though with my demos for being short on time (easy interface to use).&nbsp; i then rethought that idea and decided a pluggable model would be better.</p>
<p>alas, <a href="http://codegallery.gotdotnet.com/geocodeprovider">GeoCodeProvider</a> is born.&nbsp; in crude form right now, it seemed to work on my known tests ;-).&nbsp; i&rsquo;ve created a space for it on <a href="http://www.gotdotnet.com/">gotdotnet</a> to give my man <a href="http://blogs.msdn.com/korbyp/">korby</a> some props.</p>
<p>i&rsquo;ve also included a YahooGeoCodeProvider as an initial provider.&nbsp; the zip is an asp.net 2.0 application using atlas to demonstrate mapping the northwind (US) customers to a virtual earth map.</p>
<p>check it out: download <a href="http://codegallery.gotdotnet.com/geocodeprovider">GeoCodeProvider sample</a>.&nbsp; i&rsquo;ll soon be adding a mappoint/msn provider when i get around to it.</p>
