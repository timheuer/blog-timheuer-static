<p>i hopped on a plane at 4am this morning to head back to phoenix from seattle.  oddly enough i landed in rain.  what gives phoenix?  actually it is a welcome refresh for me.</p>
<p>i was in seattle for the <a rel="tag" href="http://silverlight.net">silverlight</a> fire starter event with my comrade <a href="http://blogs.msdn.com/mithund">mithun</a>.  this was basically a one-day silverlight lovefest.  and the lineup of people was awesome:</p>
<ul>
    <ul>
        <li><a rel="tag" href="http://silverlight.net/blogs/jesseliberty">jesse liberty</a> </li>
        <li><a rel="tag" href="http://www.adamkinney.com/">adam kinney</a> </li>
        <li><a href="http://blogs.msdn.com/webnext">laurence moroney</a> </li>
        <li><a rel="tag" href="http://www.adamnathan.net/">adam nathan</a> </li>
        <li><a href="http://blogs.msdn.com/ebooth">ernie booth</a> </li>
        <li><a rel="tag" href="http://ux.artu.tv">arturo toledo</a> </li>
    </ul>
</ul>
<p>it was awesome.  if you had <strong>silverlight</strong> questions, this was the place to be.  i mainly lurked in the back and <a href="http://twitter.com/timheuer">twittered</a> occasional thoughts on what i was seeing (which probably did nothing since a poll to the audience showed not many people are on <strong>twitter</strong> in that area).  i saw a lot of people ask the presenters questions which i assume is always a good sign.  i learned some new stuff myself which is always great.</p>
<p>adam kinney proved no smoke and mirrors on his...er...i mean...lenny's <a href="http://adamkinney.com/blog/279/default.aspx"><strong>xbox live</strong> gadget</a> built in silverlight.  after getting gamertags from the audience i think he suddenly realized that the statuses would be offline because they were all there :-) -- but yet, one person had a "1 minute ago" update and they were in the audience...hmmm...portable xboxing?</p>
<p>the <a href="http://www.popfly.com">popfly</a> demonstration seems to always get a smile from most even if you aren't interested in mashups or anything.  i think it is because you just realize the ease of bringing information together and not having to spend a ton of time together -- and getting an interesting visualization!  <a href="http://www.adamnathan.net/">adam</a> actually whipped up a similar xbox live gadget in a few minutes using popfly (not the same features, but for not writing a lick of code and getting xbox live gamer information and a silverlight UI plus implemented in a vista sidebar gadget that was impressive).</p>
<p>there was a couple of things i twitter'd about that might be worth mentioning here as well (aside from my happiness to chocolate milk on campus)...</p>
<ul>
    <ul>
        <li>arturo demonstrated the desginer/developer workflow of </li>
        <li><a href="http://www.amazon.com/dp/0735625395?tag=lamoswenebl-20&amp;camp=14573&amp;creative=327641&amp;linkCode=as1&amp;creativeASIN=0735625395&amp;adid=1BQXA25EN2FZYYQ14EY5&amp;">laurence's book on silverlight 1.0</a> </li>
        <li>there was a question about positioning text within elements.  although silverlight 1.0 does not have justification settings in elements explicitly, you can do it.  the code in my <a href="http://timheuer.com/blog/archive/2007/08/19/implement-user-control-in-silverlight-user-controls.aspx">user controls in 1.0 sample</a> has some code that you can look at how i used some simple functions to center the text on a button based on the text and size of the button...very simple.  hoping this will be obsolete in silverlight 2.0. </li>
        <li>adam kinney talked about user control patterns and specifically called out an important <a href="http://msdn2.microsoft.com/en-us/library/bb412361.aspx">createFromXaml</a> parameter (second, optional parameter) which is createNameScope.  this boolean parameter provides a unique name to each element that you create.  this is very helpful (and required almost) when you are creating a lot of same controls in an iteration for example. </li>
        <li>i learned of two tools JSLint and <a href="http://javascript.crockford.com/jsmin.html">JSMin</a>.  the latter is for 'crunching' javascript to reduce size of the script files (you know remove whitespace, etc.). </li>
        <li><a href="http://www.amazon.com/Silverlight-1-0-Unleashed-Adam-Nathan/dp/0672330075">adam nathan's book on silverlight 1.0 unleashed</a> </li>
        <li>ernie had really good pointers on working with media, specifically with what attributes in playlists are supported.  i couldn't capture them all in time, but the content will be posted. </li>
        <li>ernie also had a session on what silverlight might look like in the future...his message: understand <a href="http://windowsclient.net/learn/videos_wpf.aspx">WPF</a>. </li>
    </ul>
</ul>
<p>great sessions.  the content was recorded, so subscribe to this blog and i'll update a link once it is available.</p>
