<p>In the early days of <a href="http://silverlight.net">Silverlight</a> 2, one of the included controls was the <strong>WatermarkedTextBox</strong>.  In Beta 2 the control was removed from the runtime for among other things, WPF compatibility.  The control source code was made available for people to look at as it was referenced in a few projects, namely <a href="http://weblogs.asp.net/scottgu/pages/silverlight-2-end-to-end-tutorial-building-a-digg-search-client.aspx">ScottGu’s Digg sample application</a>.  There was hinting about future plans but nothing concrete.  </p>  <p>Now that <a href="http://timheuer.com/blog/archive/2008/10/14/silverlight-2-released-officially.aspx"><strong>Silverlight 2 is released</strong></a>, the control is not there :-).  I’ve gotten enough requests about it that I wanted to provide some action for you.  First, let me say that there are no concrete plans for WatermarkedTextBox at this time that I have to share.  I’m not saying there <em>isn’t</em> plans, but just that none at a point to make any worthwhile announcements.  Perhaps something like this would show up in the Silverlight Toolkit, but at this time no plans have been defined.</p>  <p>Again, as I’ve received enough requests, <strong>I’ve modified the source code for WatermarkedTextBox for Silverlight 2 release and am making it available for you to consume</strong>.  The code has been updated for use in Silverlight 2 and also updated to reflect the default styling of the released controls (namely to match TextBox).</p>  <p>If you aren’t familiar with the concept, it’s basically a TextBox with a value you can set for some helper text if the value is not set.  Typically you use this when you want to provide “prompt” text for some input, as an example here:</p>  <p><img src="http://s3.amazonaws.com:80/timheuer-img/watermarksample1.png" /></p>  <p>As you can see the TextBox provides a '”faded” default text that isn’t the TextBox.Text property of the control, but rather the Watermark property of this control.  Once data is entered in the control, the watermark goes away.  As long as data is input the watermark isn’t displayed…if the value is empty, the watermark displays.  Simple enough.</p>  <p>The XAML for the above looks like this:</p>  <div style="border-bottom: gray 1px solid; border-left: gray 1px solid;
