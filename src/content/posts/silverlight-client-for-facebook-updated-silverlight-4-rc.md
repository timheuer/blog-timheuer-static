<p>If you installed the <a href="http://bit.ly/facebookclient"><strong>Silverlight Client for Facebook</strong></a>, and also upgraded to the release candidate for Silverlight 4, you may have noticed it stopped working :-).</p>  <p>NOTE: Applications compiled on Silverlight 4 beta will <strong>not</strong> work on machines with Silverlight RC runtime.  This is known/expected.  As with all pre-release software, this type of breaking can be expected.</p>  <p>We’ve recently updated the Facebook application, and you will have to re-install.  Follow these steps:</p>  <ol>   <li>Uninstall the Silverlight Facebook client.  You can do this in a few ways.  First on Windows through the <em>Add/Remove Programs</em> or by right-clicking on the application while it is running (even though it doesn’t work) and choose ‘Remove Application’ – I recommend the Add/Remove Program control panel option.  On Mac OSX, simply move the application to the trash. </li>    <li>Ensure you have Silverlight 4 RC installed.  If you don’t the Facebook client welcome page will provide the links.  Reminder that this is <em>still</em> pre-release software and future uninstalls might be required. </li>    <li>Visit <a href="http://bit.ly/facebookclient">http://bit.ly/facebookclient</a> and follow the instructions. </li> </ol>  <p>You should now have the updated Silverlight for Facebook application installed.  The team received a lot of good feedback they look at.  This build doesn’t necessarily have any of those suggestions/fixes…and is more of a compatible build for the Silverlight RC runtime.  There are a few things that we finally brought forward from the initial PDC09 demonstration.</p>  <p>Custom Window options are clearly visible.  You’ll notice the ‘window chrome’ (as it is referred to) is gone and the custom window is in the application:</p>  <p><a href="http://bit.ly/facebookclient"><img style="border-right-width: 0px; margin: 0px auto; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; border-left-width: 0px" title="Silverlight Client for Facebook" border="0" alt="Silverlight Client for Facebook" src="http://storage.timheuer.com/facebookclient/fbrc-newhome.png" /></a></p>  <p>Notice the custom maximize, minimize and close buttons:</p>  <p><img style="margin: 0px auto; display: block; float: none" title="Custom window chrome toolbar" alt="Custom window chrome toolbar" src="http://storage.timheuer.com/facebookclient/fbrc-windowtools.png" /></p>  <p>Also notice in the lower right corner the resize adorner:</p>  <p><img style="margin: 0px auto; display: block; float: none" title="Resize adorner" alt="Resize adorner" src="http://storage.timheuer.com/facebookclient/fbrc-resizer.png" /></p>  <p>The updated application also implements the ‘mini-mode’ (from the toolbar area in the upper right) which produces a stripped down view of the main news feed:</p>  <p><img style="margin: 0px auto; display: block; float: none" title="Facebook Client mini-mode" alt="Facebook Client mini-mode" src="http://storage.timheuer.com/facebookclient/fbrc-minimode.png" /></p>  <p>Again this is a subtle update to keep the client working for you.  These features use the custom window options available to Silverlight 4 developers.  You can <a href="http://timheuer.com/blog/archive/2010/03/15/whats-new-in-silverlight-4-rc-mix10.aspx#custom-window-chrome"><strong>read about this updated feature</strong> here</a> and also <a href="http://silverlight.net/learn/videos/all/custom-window-chrome"><strong>watch a developer video tutorial</strong> on how to implement similar functions in your application here</a>.</p>  <p>Have fun on Facebook and hope this helps! </p>  <div style="padding-bottom: 0px;
