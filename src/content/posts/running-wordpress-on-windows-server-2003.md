<p>A few days ago, I saw someone posted on <a href="http://twitter.com/timheuer">Twitter</a> a question about running <strong><a href="http://wordpress.org/">WordPress</a></strong> on Windows Server.  Since I had done this for a few sites, I responded with my thoughts and tips.  Another suggested that I post those here, and so here I go.</p>  <p>WordPress is a blogging/content-management platform that has been around for a while.  It has been gaining more in popularity for general purpose content sites over the past year I’d say, but is primarily seen as a blogging platform by most.  Even though I use <a href="http://timheuer.com/blog/archive/2009/02/09/my-blogger-toolbox-tools-for-blogging.aspx">Subtext for my platform</a>, I have to admit that if I hadn’t already started using Subtext, it would be really hard to sway me away from WordPress.  It’s just the killer app for blogging.  Why?  Because of the immense ecosystem of people using it, there is pretty much a plethora of plugins, themes and tools for it.</p>  <p>WordPress is a PHP application and mostly used on Linux environments.  But guess what—PHP has been running on Windows for years.  In fact, our IIS team created a FastCGI implementation for IIS to make PHP applications even run better on Windows.  So here’s how I got started…</p>  <h2>Install WordPress</h2>  <p>First you obviously need to install the pieces.  I’m assuming here you have access to the actual server.  WordPress requires PHP, MySQL and the WordPress bits themselves.  You can certainly hunt around for those yourself, or you can use the <strong><a href="http://www.microsoft.com/web/wordpress/">Web Platform Installer</a></strong> from Microsoft and install WordPress on your box in about 15 minutes including download time.  No hunting for the right bits, no tinkering for the right configs.  Just install WebPI, select WordPress and go.  Honestly, this is the best way to get it started on Windows.</p>  <p>After installing you should be up and running with a starting site.  Login to complete the process.</p>  <h2>Decide on your permalinks or ‘pretty/clean URLs’</h2>  <p>Here’s the main issue with WordPress on Windows.  By default Windows 2003 is still using ISAPI filters to process requests.  What this means is that it needs something to trigger the subsystem it will use to render.  In this case it’s <em>.php</em> for the php files of WordPress – IIS then knows to use PHP to render the files/logic.</p>  <p>However because of this the default pattern of WordPress URLs on Windows 2003 will be &lt;domain&gt;/?page_id=12345.  For everything.  That isn’t very clean or friendly according to the SEO experts.  </p>  <blockquote>   <p>Now if you are installing WordPress on Windows 2008 and IIS7 the following <em>does not apply to you</em>.  IIS7 actually has built-in support for this and WordPress current versions recognize IIS7 installs and do the appropriate configuration for you!!!</p> </blockquote>  <p>If you want your URLs to be like &lt;domain&gt;/my-cool-page-title-name then you’ll have to do some hacking.  You can do a ton of high-tech stuff including actual new ISAPI filters to do some URL rewriting.  I chose the low-tech way: 404 handler.  You see if someone requests &lt;domain&gt;/some-page-name IIS won’t be able to find it and will generate a 404 message.  Luckily YOU control that 404 message.  I use a <em>very simple</em> 404 handler that I found online and modified ever-so-slightly.  You can download mine here: <a href="http://storage.timheuer.com/wp-404-handler.renametophp">wp-404-handler.renametophp</a>.  Simply put this in your WordPress root folder with all the other PHP pages.  Now in IIS Manager you’ll want to configure the IIS site to use your new custom 404 handler.  </p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="WordPress 404 Handler Config" alt="WordPress 404 Handler Config" src="http://storage.timheuer.com/wordpress-404-config.png" /></p>  <p>Just click the current one, choose URL and then type /wp-404-handler.php in there and you’re good to go.  Now you can go to the Permalinks section of the WordPress admin and choose “friendly” URLs.  Here’s what I typically use:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="WordPress Permalink Config" alt="WordPress Permalink Config" src="http://storage.timheuer.com/wordpress-permalink-config.png" /></p>  <p>Now anytime someone requests a friendly URL it will actually hit your 404 handler first, determine the right place and render the content – but keep the friendly URL in tact to the user!</p>  <p>At this point you can close your remote server session – all other routine tasks can be done via the web admin console.</p>  <h2>Find Awesome Plugins</h2>  <p>Finding plugins is VERY easy in WordPress – it’s in the administrative console of the app:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="WordPress Plugin Search" alt="WordPress Plugin Search" src="http://storage.timheuer.com/wordpress-plugin-search.png" /></p>  <p>Search for what you want and you can install <strong><em>right from the admin console</em></strong> – no need for FTP and unzip, etc.  Here is a list of my default plug-ins that I install (all of which you can search via your WordPress installation for more details and to directly install):</p>  <ul>   <li>Akismet – comment spam tool</li>    <li>All-in-One SEO pack – tools for enhancing your SEO metadata for your posts/pages</li>    <li>Contact Form 7 – a typical contact form engine</li>    <li>Google XML Sitemaps – an auto generator of sitemap.xml files for search engine indexing</li>    <li>Internet Explorer 6 Upgrade warning – you know, to support the death to IE6 cause :-)</li>    <li>Really Simple Captcha – for contact forms to prevent robots.</li>    <li>WPTouch iPhone Theme – if you want an iPhone friendly theme</li>    <li><a href="http://timheuer.com/silverlight-for-wordpress">Silverlight for WordPress</a> – to embed any Silverlight XAP in a post/page</li>    <li><strong><a href="http://timheuer.com/silverlight-for-wordpress">Silverlight Media Player for WordPress</a> </strong>– uses the Silverlight Media Framework for embedding video or IIS Smooth Streaming content.</li> </ul>  <p>There are TONS more, but these are my default that I always put on every site first.  If you want Twitter and other types of integration, they are there…just search for them.  There are currently over 9,500 plugins to choose from…amazing!</p>  <h2>Find Awesome Themes</h2>  <p>Whatever you do, don’t skip this step.  Don’t keep the default WordPress theme – it’s like any other <em>default</em> user interface thing on any software…it sucks.  Like plugins, there are a plethora of places to get awesome themes.  Here’s my default go-to list:</p>  <ul>   <li><a href="http://www.woothemes.com">WooThemes</a> – these guys are sweet.  Their menu system is actually going to be a part of WordPress 3.  Most of their themes are premium paid themes, but they do have a few free.  If you want high quality, check out these guys.</li>    <li><a href="http://www.themeforest.net">Themeforest.net</a> – a larger selection of theme from various user-contributed designers and developers.  These aren’t free either, but for &lt; $40 you can have a great quality looking theme.</li>    <li><a href="http://www.wpsnap.com">wpsnap.com</a> – these are free themes with community licenses.  There are some gems in here, but most are lower quality compared to the previous two recommendations.</li>    <li><a href="http://wordpress.org/extend/themes/">WordPress theme gallery</a> – explorable from the WordPress admin page in your installation there are a few in here (some repeats from wpsnap.com).  Usually aren’t the best to be honest.  My favorite for a cleaner default look is <em>Journalist</em> – it’s very basic, but clean looking and professional.</li> </ul>  <p>There is no excuse to have a crappy looking WordPress site no matter if you have no money, doing a blog, doing a content-managed site, etc. No excuse.  Spend 30 minutes and find a good template.  Guess what, it installs from the web admin console as well!!!</p>  <h2>Author Content</h2>  <p>While WordPress uses a web-based interface for authoring content, it isn’t required.  <a href="http://writer.live.com"><strong>Windows Live Writer</strong></a> understands the WordPress API for both posts and pages and can be used as a rich client tool for authoring content.  It’s awesome together and I use it exclusively to author content for my WordPress sites. </p>  <h2>Gotchas for running on Windows 2003</h2>  <p>Ok, while I’m sure there are others, I really only found one big <em>gotcha</em> while running WordPress on Windows 2003 (that shouldn’t apply to the IIS7/Windows 2008 installations).  This involves plugins that do form posting.  A good example is a PayPal plugin that is a complete shopping-cart for PayPal (yes it’s in the plugin gallery and free).  Most plugin developers assume your installation is likely Linux.  If this assumption is made, postbacks for some plugins might not work if you use the clean/friendly URL 404 handler I talk about above.</p>  <p>Each plugin is going to be different, but you’ll likely have to modify the post action of that plugin to point more directly to that index.php?page_id=XXXX where your content page is.  I’ve only had to do this twice and It’s relatively painless.  One thing to keep in mind though is that as plugins update (you are alerted in your web console if a new version exists), your changes will likely be overwritten, so document them!</p>  <h2>Extend WordPress yourself!</h2>  <p>Since I’m a <a href="http://www.silverlight.net"><strong>Silverlight</strong></a><strong> </strong>guy, I wanted to make sure there were some good Silverlight plugins in the gallery.  The plugin developer process is very PHP driven so if you don’t know that, follow the CASE methodology (Copy Always Steal Everything).  You can learn from others.  I created my <strong><a href="http://timheuer.com/silverlight-for-wordpress">WordPress plugins for Silverlight</a> </strong>very quickly.</p>  <p>Microsoft is also doing more with WordPress through the MIX labs team.  Check out their information on <a href="http://wordpress.visitmix.com">http://wordpress.visitmix.com</a> for other tips and how, if you want, to run WordPress on SQL Server as well!</p>  <h2>Summary</h2>  <p>WordPress is awesome.  WordPress on Windows is also awesome.  Windows 2003 takes an additional tweak, but other than that it is very simple to get running and take advantage of this ecosystem.
