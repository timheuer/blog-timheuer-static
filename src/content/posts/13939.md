<p>okay, not entirely true.  i recently <a href="http://timheuer.com/blog/archive/2006/09/14/13442.aspx#4577">got a comment</a> on how <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> might not make it in a company because one of the features, a rich editor for html that is used in rich edit areas, wiki, blog, etc., does not support firefox as a browser.  basically you can still edit, but you don't get the WYSIWYG environment.</p> <p>well, there are a couple of options.  first, the hack (i haven't even tried this to be honest so i shouldn't even be announcing it :-)).  in web.config (since sharepoint 2007 is an asp.net app) you can add &lt;browserCaps&gt; elements to the configuration section to "help" asp.net realize you have a modern browser.  <a href="http://slingfive.com/pages/code/browserCaps/browserCaps_tabs.txt">slingfive.com provides a suggested file</a> you can download and add in your web.config (please back up any web.config before you modify and test in a non-production environment).</p> <p>or...the better option.  our friends at telerik are providing and editor alternative for licensed sharepoint users.  you can <a href="http://www.telerik.com/documents/Editor_OSPS_PR.pdf">read their press release</a> to learn about the deal and you can <a href="http://www.telerik.com/products/sharepoint/overview.aspx">sign up for the current beta</a>.  they are providing an installation experience that is one-click so you don't have to mess with settings, etc.  the support for this would come from telerik.  this is great of them to do and they've done it in the past with commerce server (as pointed out in their press release).</p> <div class="wlWriterSmartContent" id="0767317B-992E-4b12-91E0-4F059A8CECA8:9e3dac2a-e38a-4ab8-9195-0a543f8284bc" contenteditable="false" style="padding-right: 0px; display: inline; padding-left: 0px; padding-bottom: 0px; margin: 0px; padding-top: 0px"></div>
