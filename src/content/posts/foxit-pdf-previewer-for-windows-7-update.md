<p>Public service announcement – if you don’t use my <strong><a href="http://timheuer.com/foxit">Foxit PDF Previewer</a></strong>, please move along…there is nothing to see here :-).</p>  <p>When<strong> Windows 7 </strong>Beta came out (build 7000), there was indeed a confirmed issue where the Foxit PDF Previewer would not work at all on 64-bit versions.  There were no install warnings, it just wouldn’t work.  It turns out this was a Windows 7 beta bug for 64-bit (some COM registering stuff if you care to know).  This was found immediately and was fixed in later Windows 7 builds (I confirmed internally with the team and had been testing daily builds).</p>  <p><strong>NOTE: This issue should have only affected 64-bit versions of Windows 7 beta.  32-bit versions were reporting unaffected.</strong></p>  <p>If you will be testing out the <a href="http://windowsteamblog.com/blogs/windows7/archive/2009/04/24/windows-7-release-candidate-update.aspx">Windows 7 Release Candidate</a> for 64-bit, I can assure you that Foxit PDF Previewer will run fine on that build (I’m running it now).  If you choose to install the release candidate on top of the beta, you may run into issues where it thinks it still isn’t working.  I recommend if you are going to go this route, <em>uninstall</em> the PDF Previewer and then run the install again.  This will correct the registering of the software correctly.</p>  <p>I hope this helps!  The download link for the Foxit PDF Preview handler is the same as the Vista one <a href="http://timheuer.com/foxit">located here</a>.</p>  <div style="padding-bottom: 0px; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; float: none; padding-top: 0px" id="scid:0767317B-992E-4b12-91E0-4F059A8CECA8:d00a1557-2a60-437b-b58b-de7ccfce421d" class="wlWriterEditableSmartContent"></div>
