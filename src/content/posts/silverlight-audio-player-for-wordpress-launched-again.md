<p>As a follow-up to my <a href="http://timheuer.com/silverlight-for-wordpress"><strong>Silverlight For WordPress</strong></a> plugin, I have just deployed Mark Heath’s <a href="http://slaudioplayer.codeplex.com/">Silverlight Audio Player</a> as a WordPress plugin.  Thanks to Mark for changing his license to <a href="http://opensource.org/licenses/ms-pl.html">Ms-PL</a> so that I could make this happen.  If you are a WordPress user and want to use Silverlight for audio playback, you use some WordPress macro language (in this case <em>slaudio</em>) and put it in.  The rendered output is like this:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Silverlight Audio Player for WordPress (collapsed)" alt="Silverlight Audio Player for WordPress (collapsed)" src="http://storage.timheuer.com/slaudio-screenshots-1.png" /></p>  <p>And when you click the play button it expands:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Silverlight Audio Player for WordPress (expanded)" alt="Silverlight Audio Player for WordPress (expanded)" src="http://storage.timheuer.com/slaudio-screenshots-2.png" /></p>  <p>Again, most of the actual <a href="http://silverlight.net"><strong>Silverlight</strong></a> work is Mark’s, so be sure to head on over to his project to thank him or if you find issues in the Silverlight player.  I’ll update the plugin whenever a new version of the player is released.</p>  <blockquote>   <p>NOTE: A lot of people ask me if I’m running WordPress on this blog because of these plugins.  The answer is no.  I run the most awesome <a href="http://www.subtextproject.com">Subtext</a> framework here.  I do have a few other sites that I use WordPress on. </p> </blockquote>  <p>To install in WordPress, simply search for the plugin in your admin dashboard under ‘silverlight’ and you will find it listed as <em><a href="http://wordpress.org/extend/plugins/silverlight-audio-player/">Silverlight Audio Player for WordPress</a></em>.  I’ve tested it on 2.8.5 but should work back to 2.2.x.</p>  <p />  <p>Hope this helps!</p>  <div style="padding-bottom: 0px;
