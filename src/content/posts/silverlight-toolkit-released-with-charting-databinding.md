<p>Today at <a href="http://www.microsoftpdc.com">Microsoft PDC</a>, Scott Guthrie demonstrated some of the new controls that have been provided as a part of the <a href="http://www.codeplex.com/Silverlight">Silverlight Toolkit</a>.  </p>  <blockquote>   <p>The <strong>Silverlight Toolkit</strong> was what was previously named the Silverlight Control Pack in the press release when the runtime was released.  </p> </blockquote>  <p>This toolkit provides a set of controls and themes for <a href="http://silverlight.net">Silverlight</a> 2.  In this initial release, they are:</p>  <ul>   <ul>     <li>AutoCompleteBox </li>      <li>Chart </li>      <li>DockPanel </li>      <li>Label </li>      <li>Expander </li>      <li>TreeView </li>      <li>UpDown </li>      <li>ViewBox </li>      <li>WrapPanel </li>      <li>ImplicitStyleManager </li>      <li><a href="http://www.codeplex.com/Silverlight/Wiki/View.aspx?title=Silverlight Toolkit Overview Part 3&amp;referringTitle=Home">Themes</a> (6): ExpressionDark/Light, RanierOrange/Purple, ShinyBlue/Red </li>   </ul> </ul>  <p><img src="http://s3.amazonaws.com:80/timheuer-img/ControlsPPC.gif" /></p>  <p>These controls are released as an initial version, with tests and available with <a href="http://www.opensource.org/licenses/ms-pl.html">Ms-Pl</a> licensing on Codeplex.  Of particular interest to me is the Charting components.  Typically available at cost with other platforms, the Silverlight team is providing a base set of Charting controls with source for you to implement, customize and ship with your products.  What is great about this type of control is an implementation with data binding as well, especially with an ObservableCollection.  With this combination you can have a powerful real-time experience updating information without having to re-paint the entire chart or do any postbacks for the app as well.</p>  <p>Let’s take a quick simple sample of what I’m talking about.  To use any of the controls in the toolkit, simply add a reference to the assembly in your project and then use the xmlns declaration at the root of your implementation.  I’m going to use “toolkit” as mine like this:</p>  <div style="border-bottom: gray 1px solid; border-left: gray 1px solid; padding-bottom: 4px; line-height: 12pt; background-color: #f4f4f4; margin: 20px 0px 10px; padding-left:
