<p>i ran across a potential snag some of you may be experiencing in visual studio web developer release version.</p>
<p>one of the goals with vs2005 was &ldquo;intellisense everywhere&rdquo; &mdash; and it was incorporated everywhere, even web.config files.&nbsp; here in lies the issue.</p>
<p>create a new web.config file &mdash; by default it is created correctly and you should have intellisense.&nbsp; save it and close it.&nbsp; now open the web site administration tool (Website menu&hellip;ASP.NET Configuration) and make some changes.&nbsp; go back to your web.config file and you may notice that the root node &lt;configuration&gt; now has an xmlns in it (which is the namespace for the config environment).&nbsp; </p>
<p>now, try to use intellisense in that modified web.config.&nbsp; does it work?&nbsp; if so, great.&nbsp; if not, you hit the snag i did &mdash; i&rsquo;ve been able to repro it but haven&rsquo;t been able to isolate if i&rsquo;m the only one.&nbsp; the workaround is to remove the xmlns attribute on the configuration node to get the intellisense back&hellip;which leads me to potential bug #2.</p>
<p>put your web.config in source control and check it in.&nbsp; close the file.&nbsp; now go back to the web site administration tool &mdash; do you get a message saying you need to have web.config be editable?&nbsp; or does the file get automatically checked out (because of your settings)?&nbsp; the WSAT tool for some reason needs the web.config to be editable (i don&rsquo;t think it does, it just thinks it does) all the time, even when you aren&rsquo;t modifying web.config properties.&nbsp; this is an annoying issue, but the workaround (if no changes are made to web.config) is to either (1) undo checkout (if you don&rsquo;t want erroneous source control audits) or (2) checkin the file again.&nbsp; again, this is annoying because of bug #1.</p>
<p>i want to point out these are DESIGN TIME issues only &mdash; no runtime execution is affected (with or without the xmlns, the web.config is readable).&nbsp; if they are annoying to you and you want them fixed, please submit your vote for the existing ladybug requests.</p>
