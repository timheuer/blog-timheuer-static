<p>The feedback from the <a href="http://timheuer.com/blog/archive/2010/05/17/silverlight-4-tools-released-and-new-application-templates.aspx">Silverlight 4 application themes released</a> and the <a href="http://timheuer.com/blog/archive/2010/06/01/sneak-peak-silverlight-application-theme-second-round-jet-pack.aspx" target="_blank">latest in process</a> have been overwhelmingly toward the positive.  We appreciate the feedback and hopefully you appreciate the transparency in the process.  As a developer I want my <a href="http://timheuer.com/blog/archive/2009/10/19/a-plea-to-my-developer-brethren-about-design.aspx" target="_blank">fellow brethren to appreciate good design</a> and use it whenever possible … even as a default if you don’t have designers on board.</p>  <p>In the initial release we had some issues getting the RIA Services ones out at the same time but we’ve got those finished now for <a href="http://www.silverlight.net">Silverlight</a> 4 Business Application Template.  </p>  <p><a href="http://go.microsoft.com/fwlink/?LinkId=192411"><strong>Download the refreshed Silverlight 4 themes (inlcuding RIA Services templates) here</strong></a><strong>.</strong></p>  <blockquote>   <p>NOTE: If the page still says version 1.0, then refresh a while or wait a bit.  Seems our servers have been having a bit of caching issues lately.</p> </blockquote>  <p>At the link above you’ll find 3 files to download:</p>  <ul>   <li>README_FIRST.txt – please read this :-) but basically I’m writing the same thing here.</li>    <li>SL4Themes-templates.zip – this includes a folder for Expression Blend and Visual Studio templates.  The VS folder also has a sub-folder for the RIA Services templates.</li>    <li>SL4Themes-rawassets.zip – this is another (optional) zip that includes the resource dictionaries for each template on their own without any Silverlight project.</li> </ul>  <p>We had an issue with the VSIX format (VSIX is the community installer format for Visual Studio to make deployment of things like templates a bit easier) and the RIA Services templates, so for now it is a manual copy process (see the README_FIRST file for location).  Thanks to <a href="http://blogs.msdn.com/corrinab">Corrina</a>, Tsitsi and <a href="http://blogs.msdn.com/deepm">Deepesh</a> for their help in refreshing these templates!</p>  <p>ETA on the new template will be a few weeks.  I’ll post updates of the progress likely on my <a href="http://twitter.com/timheuer">Twitter</a> feed.</p>  <p>Hope this helps!   </p><div style="padding-bottom: 0px; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; float: none; padding-top: 0px"
