<P>in a recent commentary by <A href="http://www.microsoft-watch.com/">mary jo foley </A>regarding <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> (<A href="http://www.microsoft-watch.com/article2/0,1995,1760375,00.asp">gates pins hopes on <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a></A>), she notes:</P>
<BLOCKQUOTE dir=ltr style="MARGIN-RIGHT: 0px">
<P>Microsoft officials have characterized <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> as "the killer app for XML."</P></BLOCKQUOTE>
<P>wow, that is a lot to swallow...and i'm not sure i could agree...at all.&nbsp; i've been working with <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> for a while (even it's predecessor portal server v1 (aka beta) and team services) as well as being a part of a joint developer program project with a client for&nbsp;the current version.&nbsp; <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> is great.&nbsp; i like it.&nbsp; it has a lot of value for a lot of organizations right out of the box.&nbsp; yeah, okay some things are weird about it and some things require a little more heavy lifting.</P>
<P>but, the &#8220;killer app for XML&#8221;?! -- seriously...whomever those microsoft officials are, should sit down and program against the web services for <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a>...while some provide some great features, the implementation of the web services is, well, odd.&nbsp; got a field named &#8220;First Name&#8221;?&nbsp; well you may not know it, but it is really named &#8220;ows_First_0x200_Name&#8221; and if your names are really long, your fields are truncated and start getting 0-based index names on them at some point.&nbsp; i understand some of the reasons for this (would have been nice for a better section in the SDK to identify these anomalies), but it isn't the best implementation that could come out of the <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> web services in my opinion.&nbsp; don't even get me started if you've renamed your field...</P>
<P>anyhow, i just thought this comment was far out there...</P>
