<p>I searched and didn’t immediately find anything, so I’m partly doing this for my own mental archive but also for others who might want this feature.</p>  <p>First, I use two tools for snapshot capturing.  When I want a quick snapshot with no annotation, I use <a href="http://www.ntwind.com/software/winsnap.html">WinSnap</a>, which rocks the hizzouse.  For when I need annotation (arrows, highlighting, etc.) I use <strong>SnagIt</strong> from the makers of <strong>Camtasia</strong> (<a href="http://visuallounge.techsmith.com/">Betsy Weber</a> rocks).  Both of these tools are incredible and I’ve purchases full licenses for both and haven’t regretted it at all.</p>  <p>In <strong>WinSnap</strong>, there is a feature that allows you to specify a tool you can run on the image that was just captured.  I’ve customized this using <a href="http://advsys.net/ken/utils.htm">PNGOut</a> based on recommendations and feedback.  Most programs create PNG files that can still be compressed by stripping out interlacing or whatever.  You can read all about image formats from <a href="http://www.codinghorror.com/blog/archives/000810.html">Coding Horror here</a>.  Jeff’s tips on bandwidth savings for web sites are invaluable.  It is helpful and works well in WinSnap.  One thing I can’t figure out is that WinSnap never <strong>saves</strong> my custom commands…that is quite frustrating, but I get over it…I don’t reboot often so I deal with it.</p>  <p>When using SnagIt though I didn’t use <strong>PNGOut</strong>.  Partly because I didn’t think it had that capability…alas but it does!  here’s how I customized it.  First, get PNGOut and put it somewhere.  For consistency with other programs I put mine in C:\Program Files\PNGOut\pngout.exe.  Yes I know there are GUIs for the tool, but I like the command-line as it integrates with this workflow.  Okay, now in SnagIt set up a new capture profile using the Add Profile wizard (I couldn’t figure out how to use SnagIt to take a capture of the SnagIt box :-) or I’d show it here).</p>  <p>Decide your settings for steps 1 and 2.  When you get to 3, do this:</p>  <ul>   <ul>     <li>Click the Properties button</li>      <li>Click the Program tab</li>      <li>Click Add and then point to where you have PNGOut.exe and save that setting</li>      <li>Under the File name area, I don’t like automatic naming so I change that to ask me for a file name</li>      <li>Click OK</li>      <li>Make sure that in the Step 3 wizard that Program is now selected (and if not select it)</li>      <li>Choose your other settings and then finish the wizard</li>   </ul> </ul>  <p>Boom, now when you capture a screenshot <strong><em>with this newly created profile</em></strong> you can annotate or do whatever you want with the capture in SnagIt and then when you click finish (after giving the file a name), PNGOut will automatically run:</p>  <p><img src="http://s3.amazonaws.com:80/timheuer-img/blah.png" /></p>  <p>This will save the image immediately, run PNGOut on it and keep it in the same place…saving you time, size and bandwidth (assuming you are uploading the image to a browser).  This works nicely with my workflow.  I think I’m going to hack together another output program that enables an immediate upload to <strong>Amazon S3</strong> so that I can capture, annotate, compress and upload in one click (well, 2).</p>  <p>Anyhow, hope this helps!</p>  <div class="wlWriterSmartContent" id="scid:0767317B-992E-4b12-91E0-4F059A8CECA8:1e39235e-732c-438a-988d-952fd9a88c34" style="padding-right: 0px;
