<p>I <a href="http://timheuer.com/blog/archive/2009/02/09/my-blogger-toolbox-tools-for-blogging.aspx">run my site on Subtext</a> which has been around for 6+ years in some form (Subtext is a fork of .Text from way back).  As a part of the framework, there was initially built-in capabilities for tracking referral traffic.  On each view of the application, it would tick a referral note and you could see this in the statistics view of the admin pages.</p>  <p>As the standards (for lack of a better term) of tracking Page Views, Referrals, etc. moved to more proven/consistent reporting like Google Analytics (or other platforms) these type of platform tracking became worthless to me.  I never checked them because, frankly, I didn’t believe them anyway.  The problem is that Subtext is still tracking this information for me and taking up valuable little bytes in my database.</p>  <p>For Subtext, specifically, contributors have created <a href="http://haacked.com/archive/2006/02/06/databasemaintenanceofyourblog.aspx">scripts</a> and <a href="http://www.paraesthesia.com/archive/2011/06/17/easier-subtext-2-5-2-0-maintenance.aspx">maintenance pages</a> to help manage some of these referrals that may not matter to folks and are just taking up space.  I am one of those people.  In my recent <a href="http://timheuer.com/blog/archive/2012/02/20/migrating-existing-sql-server-to-sql-azure.aspx">migration to SQL Azure</a> I wanted to take advantage of 100MB pricing.  Surely my blog was not bigger than that.  To my surprise my blog was <strong>650MB</strong> in size.  </p>  <p>What!?</p>  <p>I hadn’t run my database maintenance script in a while and decided to run that which purges the referral tracking.  It got down to <strong>35MB</strong>.  Yeah baby.  In fact this topic has been discussed on a few times on the Subtext developer mailing list and even tracking as a <a href="https://github.com/Haacked/Subtext/issues/21">issue for the project</a>.  In the meantime I wanted to solve it myself for my blog.</p>  <p>In Subtext there is a stored proc that runs to get some of the entry tracking data called subtext_TrackEntry.  Within that proc is where it looks to see if it is a referral and adds that data.  I simply <p>Now I’m no longer tracking referrals because my analytics package is doing that for me already.  My database is now representative of things that matter to me, rather than things I just want to clean up.  If you are a Subtext user and never knew that referral logging was wasting your database (and you are using an analytic package to track that anyway), then I hope this helps!  </p><div style="padding-bottom: 0px; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; float: none; padding-top: 0px" id="scid:0767317B-992E-4b12-91E0-4F059A8CECA8:c5b2ba25-5f89-4ae1-aa59-e3152525da4b" class="wlWriterEditableSmartContent"></div>
