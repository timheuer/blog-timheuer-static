		<p>in a <a href="http://timheuer.com/blog/archive/2006/06/15/12683.aspx">previous post</a> i started to take a quick look at the new <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> bits released this week at teched.  to recap, essentially the core functionality of <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> (i love saying that…they should have named it bling) is to generate a CRUD structure of code and asp.net pages for your data structure.  the “inq” in <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> is because the code generated uses LINQ.  okay, all that aside, let’s start taking a look.</p>
		<p>
				<a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> is a command-line tool.  type <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> to see the options.  for simplicity comparison sakes for rails, i chose the /noStyle option so it would generate the vanilla-plainjane-nofrills-whiteasmilk scaffold site on my pubs database.  yes, i’m using pubs — it’s easy to relate to most and has a simple structure including some foreign key relationships, etc.</p>
		<p>for those new to rails (ruby on rails), i’m not going to go through a tutorial, and if you don’t know/care what rails is, then move along, there is nothing to see here.  if you know a bit, then here’s a little bit more info to help setup the comparison.  after installing rails, you’ll be working with sql server (or sql server express – which is free).  to work with sql server in ruby/rails you’ll need want an additional database library for ado to make things a bit easier.  <a href="http://wiki.rubyonrails.org/rails/pages/HowtoConnectToMicrosoftSQLServer">you can download it here and read more info about it</a>.  </p>
		<p>ensure the file (ADO.rb) is located in your </p>
		<pre>&lt;rubyinstall&gt;\lib\ruby\site_ruby\1.8\DBD\ADO</pre>
		<p>
		</p>
		<p>once you have that and after you have generated your rails app go to your database.yml file and modify the configuration.  for simplicity sake, i’m only using one configuration type (development) to make things easier, since i’m just experimenting anyway.  here’s what my database.yml looks like:</p>
		<p>
		</p>
		<pre>development:<br />  adapter: sqlserver<br />  database: pubs<br />  host: (local)\SQLEXPRESS</pre>
		<p>
		</p>
		<p>after i have that, i’m ready to go.  again for simplicity sake, i’m just using the authors table.  so generate a model and a controller for author.  <strong><em>here lies my first beef with rails when compared to <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a>.  in rails you have to generate all your scaffolds individually.  unless i’m mistaken and there is an uber scaffold command, i like how <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> looks at all the data and generates them all for you.</em></strong>  <strong>*however*</strong> it would be nice to specify params maybe…so that i could have utmost flexibility — default generate all, or add an argument option for me to specify the tables.  anyhow, i digress…</p>
		<p>then in your author controller, change it to scaffold:</p>
		<p>
		</p>
		<pre>class AuthorController &lt; ApplicationController<br /> scaffold:author<br />end</pre>
		<p>
		</p>
		<p>once that is in place, and you start your WEBrick server, you should be able to browse to <a href="http://localhost:3000/author/">http://localhost:3000/author/</a> and get this:</p>
		<p>
				<a href="http://www.flickr.com/photos/timheuer/168389684/">
						<img alt="railsscaffold1 [2]" src="http://static.flickr.com/69/168389684_cdfcae9c7f.jpg" border="0" />
				</a>
		</p>
		<p>
				<strong>
						<em>and therein lies my love of rails scaffolding!</em>
				</strong>  a simple, single line in the controller and i have full access to all the features.  rails is doing everything behind the scenes.  a quick look at my application views folder will show no files…think httpmodule for you asp.netters.  <strong><em>i wish <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> would implement this — an httpmodule/handler that would allow me to configure the scaffold and then do all the magic goop for me</em></strong>.</p>
		<p>
				<em>
						<strong>here’s my second disappointment with rails scaffolding</strong>
				</em>…there is no default page…i like how <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> generates an index page for me so that i can go to the site and then pick the pages i need to play with.  even though <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> does this for me though, when on a page (i.e., authors), <strong>there is no ‘back to index’ link at all</strong>…so i rely on the back button too much.  now remember i stated earlier that i used the /noStyle option to generate my <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> bling.  if i hadn’t, then <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> *does* generate a skin/theme for me which uses the asp.net menu control and on every page i can see the nav structure.  it uses a master page.  i think even in the /noStyle option, a master page with a nav structure would be helpful — but i guess /noStyle means “no style” <img src="http://www.timheuer.com/images/smile1.gif" />.</p>
		<p>okay, so we’ve gotten this going…are you sick of command lines yet?</p>

		<p>it should be important to note that i’m calling it rails scaffolding, but technically it is rails in combination with activerecord doing the ruby/rails magic here.</p>
		<p>one thing that i think both suck at right now are two key things.  </p>
		<p>
				<strong>validation</strong>: it knows what is required, etc., so why not wire up some validation for me?  instead i get goobly-guck error messages (note to our asp.net team: we should use the customErrors and make something nice in the generated page)</p>
		<p>
				<strong>foreign key relationships</strong>: it doesn’t generate user friendly views of this.  take the employees view (shown in <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> gen here):</p>
		<p>
				<a href="http://www.flickr.com/photos/timheuer/168389700/">
						<img alt="blinqedit2" src="http://static.flickr.com/49/168389700_ba37eeb8f7.jpg" border="0" />
				</a>
		</p>
		<p>the drop downs are the values of the relationships and don’t portray the user-friendly view of the data…rails generated code is no different.  again, <em><strong>we know the relationship…let’s put some more magic in here to make it whiz-bang</strong></em>!</p>
		<p>one thing to note is that <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> does get it a bit.  look below where my cursor is:</p>

		<p>there is a link created for titleauthors — which is the relationship on this authors table…so i can directly go to that location.  in rails, this isn’t there…</p>

		<p>paging: this is a quick one — in rails, it is ‘next page’ and ‘previous page’ (just look at some of the shots above) — so how many ‘next’ pages are there?  using the gridview in asp.net gives us numeric paging — done.</p>
		<p>one thing that rails understands is plurality.  my table is actually called ‘authors’, but you’ll notice the model/controller/etc are authors — it ‘gets it’ and does everything for me (yes, people/person works too).  <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> doesn’t necessarily have this problem because it is generating specific code for the tables…if the handler/module method was implemented, maybe that could be examined…but for now it is a ‘cool’ feature in rails — not earth shattering, but nice to know if you need it.</p>
		<p>for where i think <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> misses right now is in some easy hits.  </p>
		<ul>
				<li>where’s <a href="http://atlas.asp.net/">atlas</a>?  why not make all the pages zing with some <a href="http://atlas.asp.net/" rel="tag">atlas</a> uber magic?  how about an /async option on <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a>.exe? the CRUD pages scream with pleading for some low-hanging <a href="http://atlas.asp.net/" rel="tag">atlas</a> love. 
</li>
				<li>please put in a delete confirmation 
</li>
				<li>show me my index link 
</li>
				<li>linq — this is cool how it uses this…take a moment to examine the code generated…it is pretty fun to look at — the pages using objectdatasources to bind to linq objects…yummy.</li>
		</ul>
		<p>anyhow, a quick examination of the two in a few minutes…there are pros and cons to how each generates the respective ‘scaffold’ for their technologies (i’m not arguing the merits of the actual technologies here, just comparing some like implementations of features).</p>
		<p>i think that <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> will evolve over time, hopefully more for the better — maybe (crossing fingers) <a title="BLINQ" href="http://www.asp.net/sandbox/app_blinq.aspx?tabid=62">BLINQ</a> will be another open source project out of the asp.net team — <a href="http://www.codeplex.com/" rel="tag">codeplex</a> anyone?</p>
