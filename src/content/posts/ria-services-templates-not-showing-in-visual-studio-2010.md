<p>Last week, the <a href="http://timheuer.com/blog/archive/2009/11/18/whats-new-in-silverlight-4-complete-guide-new-features.aspx">Silverlight 4 beta release</a> included the <strong>Silverlight 4 Tools for Visual Studio 2010</strong>.  This single installer would perform the following (assuming you had either Visual Studio 2010 or Visual Web Developer Express 2010 already installed:</p>  <ul>   <li>Install a Visual Studio 2010 service pack (KB976272) </li>    <li>Install Silverlight 4 Windows developer runtime (4.0.1108.0) </li>    <li>Install Silverlight 4 SDK </li>    <li>Install WCF RIA Services (November 2009) </li> </ul>  <p>That is all you really needed.  But some may have had an experience afterwards of launching VS2010 and *NOT* seeing the <em>WCF RIA Services Class Library</em> or <em>Silverlight Business Application </em>templates:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="WCF RIA Services Templates in VS2010" alt="WCF RIA Services Templates in VS2010" src="http://storage.timheuer.com/riaservicesinstallfail1.png" /></p>  <p>What is happening here is that likely you already had a version of .NET RIA Services (likely the July 2009 CTP) installed.  The Silverlight tools installer silently failed and just kept going.</p>  <h2>If you don’t see the WCF RIA Services Templates…</h2>  <p>If you don’t see the <strong><a href="http://silverlight.net/riaservices">WCF RIA Services</a></strong> templates, make sure Visual Studio 2010 and Visual Studio 2008 are both shutdown and perform the following:</p>  <ul>   <li>Go to the <em>Add/Remove Programs</em> control panel application and locate .NET RIA Services.  Select the item and uninstall it.  This will remove the previous July CTP of the artist formerly known as .NET RIA Services. </li>    <li>Re-run the Silverlight Tools for Visual Studio 2010 installer.  This will re-install the items and ensure that RIA Services is properly installed      <ul>       <li>Optionally you can extract the RIAServices.msi installer by running the tools installer with the <em>/x:&lt;folder&gt;</em> switch which will extract the contents. </li>     </ul>   </li> </ul>  <p>After performing the above, you should now see the WCF RIA Services templates and should be working fine.</p>  <h2>So, what happened?  Why no fail log?!</h2>  <p>We did say beta right? :-) -- In all seriousness, we wanted to ensure that the WCF RIA Services bits got in the tools installer and knew this little inconvenience might creep up for some.  Apologies for the inconvenience.  There also exists no logging of the failure to indicate that there was anything wrong and to assist you in knowing the situation.</p>  <p>This is being fixed in a future installer to detect a previous install of RIA Services and alert the user (or we may even force uninstall it…not sure yet).  </p>  <h2>But wait, RIA Services isn’t working in my Visual Studio 2008 environment now!</h2>  <p>That’s right.  If you install RIA Services for Silverlight 4/Visual Studio 2010, it isn’t going to work in your Visual Studio 2008 environment.  If you want to work on RIA Services with Silverlight 3 and Visual Studio 2008, then you need to stick with the <a href="http://www.microsoft.com/downloads/details.aspx?FamilyID=76bb3a07-3846-4564-b0c3-27972bcaabce&amp;displaylang=en">WCF RIA Services for VS2008</a> version.</p>  <p>Bottom line: <strong><em>RIA Services does not install side-by-side.</em></strong></p>  <p>All of this information is provided to you on the <a href="http://silverlight.net/riaservices">WCF RIA Services web site</a>.  The first section explains the bits you need for each environment.</p>  <p>Hopefully this helps clear some confusion.  Yes, I know it is frustrating not getting an error and not seeing things work.  We’re working to solve that (doesn’t help you now, I know) in future setups and make sure the release tools installer does the right thing.  It’s during these beta periods we can help identify such issues to make sure we fix them before release.</p>  <p>If you have questions on WCF RIA Services, be sure to head to the <a href="http://forums.silverlight.net/forums/53.aspx">forums</a> where the team is listening!</p>  <p>Hope this helps!</p>  <div style="padding-bottom: 0px; margin: 0px; padding-left: 0px;
