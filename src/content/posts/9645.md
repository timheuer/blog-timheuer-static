<p>conference food breakfasts usually don&rsquo;t intrigue me.&nbsp; i&rsquo;m not even a breakfast person &mdash; i hate it (yeah, most important meal of the day, blah, blah &mdash; leave me alone &mdash; i&rsquo;m fat and getting fatter, so what.)</p>
<p>but this morning was different&hellip;</p>
<p>i sat down and was listening to <a href="http://brian.behlendorf.com/" rel="tag">brian behlendorf</a> and others chat about source control and collaboration.&nbsp; it was very intriguing to hear&hellip;some felt that source control isn&rsquo;t necessary for even smaller samples sets due to the effort to even set it up.</p>
<p>i won&rsquo;t iterate through the entire conversation, but one comment i felt was interesting was about how poeple (especially the open source community) are less likely to use anything if there isn&rsquo;t an infrastructure for collaboration and evolation on that &ldquo;thing&rdquo; (read: sample, project, <a title="" href="http://www.gotcodesnippets.net/">snippet</a>, etc.).&nbsp; i completely agree with this.&nbsp; and i&rsquo;ve personally fallen into that as well.&nbsp; i&rsquo;ve submitted samples, written code and even promised distribution, but simply haven&rsquo;t done it with some things like <a title="" href="http://www.timheuer.com/downloads/rssfeedreadersetup.msi">feedreader</a>.&nbsp; i&rsquo;m at fault.&nbsp; this statement made me really realize how much better my code could be if that infrastructure was created around them.</p>
<p>one other comment that was interesting was almost the need to combine wiki-ish activity around this source collaboration&hellip;pointing out that checkin/checkout isn&rsquo;t enough.&nbsp; brian pointed out that viewvc introduces that functionality for subversion&hellip;</p>
<p>hmm&hellip;maybe <a href="http://www.gotdotnet.com/" rel="tag">gotdotnet</a> needs to implement better collaboration to be more viral? &mdash; think it will happen ;-)?</p>
