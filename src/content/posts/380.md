<P>as some of you may recall...i had a web part which was the first navigational web part developed by a third party.&nbsp; since then several others have come out with varying results and features.&nbsp; i was quite disappointed that i couldn't distribute my enhanced navigational component, but for various reasons i simply couldn't.&nbsp; i've had a version that has been &#8220;laying in wait&#8221; since september of last year (see screenshot below).&nbsp; i also have been holding onto (and already talked about it) a document library explorer...just been procrastonating in releasing!</P>
<P>one thing that most have neglected in the navigational components have been security and being able to render a complete tree both up and down (as well as integration with portal and/or wss in one common web part).</P>
<P>what i'm curious about is if people are interested?&nbsp; there seem to be a lot out there now (or maybe not a lot, but enought that do the same thing).&nbsp; i hate to seem like i'm following the crowd because now i'd be able to distribute an enhanced version of what i'd already created as the first component/web part.&nbsp; even today, there was an SPTrees (see below) sample released on gotdotnet.com.</P>
<P>anyhow, feel free to chime in.</P>
