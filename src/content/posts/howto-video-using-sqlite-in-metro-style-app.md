<p>I got a few questions and comments about how to actually include SQLite in a C# Metro style app.  Since perhaps it wasn’t clear in describing in <a href="http://timheuer.com/blog/archive/2012/05/20/using-sqlite-in-metro-style-app.aspx">my post</a>, I thought a quick video might help demonstrate the steps to build and use SQLite in a C# Metro style app.</p>
<p>The video walks through actually building SQLite from the source (Visual Studio 2012 required…express is fine) and adding it to a C# Metro style app, create a database, populate with some data based on a class and databind the query to a ListView.  The video references my <a href="http://sdrv.ms/Kz8XKV">OneNote notebook on the tools you need to download and build the SQLite source</a>.  It also demonstrates using the <strong><a href="https://nuget.org/packages/sqlite-net">sqlite-net library from NuGet</a></strong> on interacting with SQLite in a C# application.</p>
<p><iframe width="640" height="360" src="http://www.youtube.com/embed/luV6no8ti6M?rel=0&amp;hd=1" frameborder="0" allowfullscreen=""></iframe></p>
<p>This is a quick video to demonstrate the concept on how to get started and is not a full end-to-end sample.</p>
<blockquote>
<p>NOTE: This video only demonstrates how to build SQLite until the team itself merges the WinRT changes and produces the supported build.  Until then this is a step you’d have to do on your own and these private branches are not fully supported by them until merged to their main release branch.</p>
</blockquote>
<p>I hope this helps clarify things!</p>
