<p>I’ve seen some reports and received some emails on email groups that I’m a part of around Windows 8 and Google Chrome browser and how touch is not working.  In fact I was initially confused about this myself because it was working fine for me on my machine (Lenovo x230t) but other people were insistent that it wasn’t working.  Then I asked what machine they were on.</p>  <p>Almost exclusively everyone was on a “high DPI” machine (or had high DPI settings in their display).  If you have a Surface Pro, for example, you are using a high DPI machine.  I installed Chrome on my Surface Pro and indeed saw the problem there.  After some digging it appears to be that Google Chrome is not yet a high DPI-aware desktop application (at least of this writing which was using version 30.0.1599.66 m – phew that was a mouthful of a version number).</p>  <p>Look at this screenshot.  This is from my Surface Pro with no changes.  I’m tapping on the 2nd image in the news highlights (indicated by the touch square indicator) but notice where the target is actually clicking – on the date “October 4” where the context menu is located:</p>  <p><img title="Chrome missed touch target" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Chrome missed touch target" src="http://storage2.timheuer.com/chromemistarget.png" /></p>  <p>I asked around the office and received one workaround, which I received, but also found a better one.  Here are both of them for you to consider.</p>  <h2>Workaround 1: run in compatibility mode</h2>  <p>This involves you getting to the properties of the chrome.exe program and viewing the compatibility tab.  In there you can check the option to disable DPI scaling:</p>  <p><img title="Chrome compatibility setting" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Chrome compatibility setting" src="http://storage2.timheuer.com/chromecompatsetting.png" /></p>  <p>I didn’t like this option because immediately after restarting Chrome the display was crisp but “tiny” to see for my aging eyes.  You can see here is the msn.com site before/after this compatibility setting:</p>  <p><img title="Chrome msn.com before" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Chrome msn.com before" src="http://storage2.timheuer.com/chromemsnbefore.png" /></p>  <p align="center"><em>msn.com on Google Chrome with no compat setting</em></p>  <p align="center" />  <p><img title="Chrome msn.com compat set" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Chrome msn.com compat set" src="http://storage2.timheuer.com/chromecompatrefresh.png" /></p>  <p align="center"><em>msn.com on Google Chrome with the compat setting enabled</em></p>  <p>After this setting the hit-targets are fine and work but the content is small (although admittedly more of it is viewable, but that is too small for me).  To mitigate this you can use Chrome’s zoom feature to zoom to 150% to match most closely the high DPI scaling that automatically happens.</p>  <p><img title="Chrome zoom settings" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Chrome zoom settings" src="http://storage2.timheuer.com/chromezoomsetting.png" /></p>  <p>But there has to be a better way…and is (in my opinion).</p>  <h2>Workaround 2: set the Chrome flag for HighDPI Support</h2>  <p>I learned that Chrome has “flags” that you can set to experiment with certain settings or override other default/automatic settings.  From the address bar, type “chrome://flags” and you’ll see a page of options.  Scroll near the end and you’ll see a “HighDPI Support Windows” setting which is set to Automatic.  </p>  <p><img title="Chrome HighDPI Flag setting" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Chrome HighDPI Flag setting" src="http://storage2.timheuer.com/chromehdpiflag.png" /></p>  <p>Force this to Enable and re-launch Chrome.  Boom you are working again.  Simpler fix and gets you scaling on your device as well.  Now some of Chrome’s chrome (menus, etc.) aren’t as crisp still but the main content area doesn’t confuse hit-targeting of your touch interaction because of the scale.</p>  <p>The good part is that it appears that the Chrome team is aware of this and a Chrome bug is logged (<a href="https://code.google.com/p/chromium/issues/detail?id=271222">Chromium Bug 271222: When using the global Windows UI scaling feature, touchscreen touches are off by the set scale</a>).</p>  <p>I don’t use Chrome on a regular basis (nothing against it, just don’t need it) but do get sad when friends have problems so I like to try to help out.  Hopefully if you are in this case with your touchscreen and Google Chrome this will get you fixed up.</p>  <p>Hope this helps!</p>  <div id="scid:0767317B-992E-4b12-91E0-4F059A8CECA8:b07ff8d4-0980-42ad-bfe6-ea3543559ec9" class="wlWriterEditableSmartContent" style="float: none;
