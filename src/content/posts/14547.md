<p>In an email dialog today I saw someone asking how he could use an existing <a href="http://www.microsoft.com/expression/products/Overview.aspx?key=encoder">Encoder 2</a> template for existing media or streaming URIs when you don’t have something to encode.  After a few explanations, he replied that <em>someone should blog this</em> – and I agree :-).</p>
<p>So what did he mean?  Well, when you use Expression Encoder, you are typically going to be <em>encoding</em> media to a format to consume.  Encoder also gives you an additional option in the output settings to generate a media player for that encoded content.  These are all based on templates that I’ve previously written about that are available in the product as well is the <a href="http://timheuer.com/blog/archive/2008/11/03/use-expression-encoder-templates-in-silverlight-application.aspx">source code</a> for you to extend.</p>
<p>But what if you have a media file that doesn’t need encoding or you have a streaming URI and you don’t need to <em>encode</em> anything, but want the player?  To some it might not be so obvious so let me try to walk you through the steps.</p>
<p><strong><u>You need the player</u></strong></p>
<p>First you need the player.  The XAP that is, of the template you want to use.  There are essentially two ways of going about getting this:</p>
<ol>
    <ol>
        <li>Getting the MediaPlayerTemplate.xap file from an existing output you already did (that used the same template you want)</li>
        <li>Building the XAP using the source of the templates and building it from scratch (requires .NET compiler or Visual Studio)</li>
    </ol>
</ol>
<p>Obviously if you have the XAP of your desired template, you are ready to proceed.  If you don’t, then you’ll need to proceed to step 2.  Here’s what you’ll do:</p>
<p><em>Building the Player</em></p>
<p>First, find the template you want.  These are located in <em>C:\Program Files\Microsoft Expression\Encoder 2\Templates\en</em> (note: if you are on 64-bit or a different language, this path may differ slightly).  Within here you’ll see a list of templates by name.  Find the one you want and within that folder there is a <em>Source</em> directory.  That’s what you want.  <strong>I highly recommend actually moving the source files to a different place rather than edit directly in the templates directory.</strong>  There is a Visual Studio 2008 solution/project file in there and you can open it up and compile.  </p>
<blockquote>
<p>As long as you are in there…</p>
<p>If you don’t need <em>all</em> the features of the template (i.e., Adaptive Streaming, etc) consider <strong><a href="http://www.clarkezone.net/default.aspx?id=e97074d8-7b4a-4d0a-a4db-0f03a82ad3b9">reading James Clarke’s post on removing some of those references</a></strong> and still creating the same template, but with reduced functionality.</p>
</blockquote>
<p>Once compiled (remember to choose the right configuration - i.e., Release, Debug - for your needs), you’ll have in the Bin directory of that project the MediaPlayerTemplate.xap file you need.  </p>
<p><strong><u>Passing parameters</u></strong></p>
<p>So how can you compile the player template without media?  The Encoder 2 Silvelright 2 templates are completely parameter driven.  If you notice in any existing encoded project output (look at the default.html file generated) you’ll see the object tag used to host the player.  It may look something like this:</p>

<p><strong><u>Summary</u></strong></p>
<p>So having the MediaPlayerTemplate.xap file is really all you need.  Anything else can be sent as parameters either via the object tag, the XAML control or in code.  This enables being able to really re-use the template you desire without having to run an encoder job just to get the template and change the code.</p>
