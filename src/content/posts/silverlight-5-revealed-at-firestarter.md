<p><img border="0" align="left" style="background-image: none; border: 0px none; margin: 0px 10px 3px 0px; padding-left: 0px; padding-right: 0px; display: inline; float: left; padding-top: 0px;" title="Silverlight 5" alt="Silverlight 5" src="http://storage2.timheuer.com/sl5bloglogo.png" /></p>
<p>Today we had the <a href="http://silverlight.net">SIlverlight</a> Firestarter event and revealed to the world for the first time, not only the plans for the next version of Silverlight, but also a slew of demonstrations of some key features.</p>
<p>If you weren’t able to attend live or online real-time, we have recorded the sessions which will be available later this week on <a href="http://channel9.msdn.com">Channel 9</a>.</p>
<p>There has been a lot of confusion on the future of Silverlight.  Microsoft wants to ensure we have a great developer platform portfolio that supports all great technologies like HTML5, Silverlight and WPF.  We’ll continue to invest in these technologies for the future of developers to target multiple platforms, technologies and devices.</p>
<p>But for now, let me share some of the things we announced that we’re working on to deliver for Silverlight 5…</p>
<h2>Media Improvements</h2>
<ul>
    <li>Hardware Decode and presentation of H.264 media</li>
    <li>“TrickPlay” allows video to be played at different speeds and supports fast-forward and rewind. At up to twice the speed, audio pitch correction allows users to watch videos while preserving a normal audio pitch.</li>
    <li>Improved power awareness prevents the screen saver from being shown while watching video and allows the computer to sleep when video is not active.</li>
    <li>Remote-control support allowing users to control media playback.</li>
    <li>Digital rights management advancements allow seamless switching between DRM media sources.</li>
</ul>
<h2>Application Development</h2>
<ul>
    <li>Fluid user interface enables smoother animation within the UI. Layout transitions allow developers to specify animations to apply when elements are added, removed or re-ordered within a layout. This provides smoother user experiences when, for example, items are inserted into a list.</li>
    <li>Text improvements make it possible to build rich magazine-style text layouts:
    <ul>
        <li>Multicolumn text and linked text container allow text to flow around other elements.</li>
        <li>Tracking/leading set precisely how far apart each character is for full creative control.</li>
        <li>Text clarity is improved with Pixel Snapping.</li>
        <li>Text layout performance is significantly improved.</li>
        <li>OpenType support has been enhanced.</li>
    </ul>
    </li>
    <li>Support for Postscript vector printing enables users to create reports and documents, including the ability to create a virtual print view different from what is shown on the screen.</li>
    <li>Applications can now work the way users expect with added support for double-click and ComboBox type ahead.</li>
    <li>Databinding enhancements allow more work to be done more easily via XAML:
    <ul>
        <li>Debugging support now allows breakpoints to be set on a binding, so you can step through binding failures.</li>
        <li>Implicit DataTemplates allow templates to be created across an application to support a particular type by default.</li>
        <li>Ancestor RelativeSource allows, for example, a DataTemplate to bind to a property on the control that contains it.</li>
        <li>Binding in style setters allows bindings to be used within styles to reference other properties.</li>
        <li>The DataContextChanged event is being introduced. Markup extensions allow code to be run at XAML parse time for both properties and event handlers, enabling cutting-edge MVVM support. </li>
    </ul>
    </li>
    <li>Networking and Windows Communication Foundation enhancements:
    <ul>
        <li>Reduced network latency by using a background thread for networking.</li>
        <li>WS-Trust support: Security Assertion Markup Language authentication token.</li>
    </ul>
    </li>
    <li>Silverlight 5 performance improvements include these:
    <ul>
        <li>Reduced network latency by using a background thread for networking.</li>
        <li>XAML parser improvements that speed up startup and runtime performance.</li>
        <li>Support for 64-bit operating systems.</li>
    </ul>
    </li>
    <li>Graphics improvements
    <ul>
        <li>Graphics Processing Unit (GPU) accelerated 3-D application programming interface (API) provides rich graphics on the Web for building advanced data visualizations and rich user experience (UI).</li>
        <li>Immediate mode graphics API allows direct rendering to the GPU.</li>
        <li>Hardware acceleration is enabled in windowless mode with Internet Explorer 9.</li>
    </ul>
    </li>
    <li>Silverlight offers a new class of trusted applications that brings desktop capabilities to the browser for the first time. These features, when enabled via a group policy registry key and an application certificate, mean users won’t need to leave the browser to perform complex tasks:
    <ul>
        <li>Host HTML content as a Web browser control within the Silverlight application. HTML pages, such as help content or e-mail, can be integrated within the application.</li>
        <li>Read and write files to the user’s My Documents folder, making it easier to find media files or create local copies of reports. </li>
        <li>Launch Microsoft Office and other desktop programs. Users can open Microsoft Outlook and create an e-mail message, or send a report to Word utilizing the power of Office. </li>
        <li>Access devices and other system capabilities by calling into application COM components. Users can access a USB security card reader or a bar-code scanner.</li>
        <li>Enjoy full keyboard support in full screen, which enables richer kiosk and media viewing applications.</li>
        <li>Call existing unmanaged code directly from within Silverlight with P/Invoke.</li>
    </ul>
    </li>
    <li>Out-of-browser trusted applications are further enhanced:
    <ul>
        <li>Existing unmanaged code can be called directly from within Silverlight with P/Invoke.</li>
        <li>Child Window support allows multiple windows to be launched from the application.</li>
    </ul>
    </li>
</ul>
<h2>Summary</h2>
<p>So as you can see an impressive list of features we’ve been really working hard on to deliver.  We hope you like the features.  Please continue to suggest features to our team at <a href="http://silverlight.mswish.net">http://silverlight.mswish.net</a> and vote up others that already exist.  We really hope you look forward to the next version of Silverlight as we’re excited to bring it to you.</p>
<p>Sorry to burst your bubble, but no downloadable bits for you to play with just yet :-).</p>
<p>Hope this helps!   </p>
