<p>UPDATE/correction: <a href="http://jeffweber.spaces.live.com/">jeff weber</a> authored the farseer engine, bill ported for silverlight...both did good work!  thanks for the clarification!</p>
<p>bill reiss of <a href="http://www.bluerosegames.com/brg/drpopper/default.html">dr. popper</a> fame (my daughter loves the "bubble game" she calls it), just released the farseer <a href="http://www.codeplex.com/FarseerPhysics">physics engine</a>, which has support for silverlight (1.1) as well.</p>
<p>the engine is an easy to use 2D physics engine for XNA and silverlight platforms.  it contains logic for collision, joints, controllers, etc...everything you would need to start a base casual game at least.</p>
<p>bill put it out on <a href="http://www.codeplex.com/FarseerPhysics">codeplex</a> as an open source project as well released under the Ms-Pl.  check out some of the demos of the <a href="http://www.bluerosegames.com/farseersilverlightdemos">functions here</a>.</p>
