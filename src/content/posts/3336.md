<P>i&#8217;ve recently been getting more into open source technologies, specifically those that tough .net a bit in some way shape or form &#8212; mono, cocoa#, ruby, struts.</P>
<P>i&#8217;m frustrated.</P>
<P>check that&#8230;i&#8217;m pissed.</P>
<P>okay, fine, i get it &#8212; open source is cool because you always get source code and for the supergeeks, it is cool to see how things work&#8230;</P>
<P>but take someone who is a developer but hasn&#8217;t done development on alternate platforms before&#8230;wouldn&#8217;t you want an easier way to get them adopted?&nbsp; wouldn&#8217;t you want the learning process &#8212; or at least setup process &#8212; to be smooth so that adoption would be easier?</P>
<P>case in point&#8230;(for me at least): cocoa# and monodevelop on mac osx.&nbsp; figure you&#8217;d go to <A href="http://www.monodevelop.org/">www.monodevelop.org</A> and get some goodies right?&nbsp; sure.&nbsp; download and install them right? um, kinda.&nbsp; download, yes.&nbsp; install?&nbsp; well, not unless you have &#8220;slash bin&#8217;d&#8221; in a previous life.&nbsp; navigate to your monodevelop-0.5.1 directory (note: you must do this in terminal mode&#8212;if you don&#8217;t know what that is, rewind to linux programming 101&#8211;-remember, these guys don&#8217;t typically think GUI is a friendly acronym) and type &#8220;./configure&#8221; &#8212; watch all the output that you have no idea what it is doing (that&#8217;s okay&#8212;on other platforms I&#8217;m sure you&#8217;re not watching the registry GUIDS fly across thinking &#8220;ah, yes, it just installed the fitzer protocol&#8221;).&nbsp; then if configure worked, run make.&nbsp; then if make worked, run make install.&nbsp; seriously? yes, seriously.&nbsp; okay, MS may not have everything in order but i dare anyone to say we allow anyone to get going fast on ANY of our products&#8230;dare i say even betas&#8230;at least we provide a method to install it for the brainless (most of the time).</P>
<P>so i&#8217;ve not successfully got cocoa# and monodevelop on mac osx running&#8230;and a quick google/msn/coolest-ajax-site-of-the-day search tells me i&#8217;m not the only one&#8230;several have given up.&nbsp; others keep trying.&nbsp; monodevelop&#8217;s site provides some &#8220;instructions&#8221; that apparently the community has invalidated, but guess what &#8212; nobody is stepping up to create a correct one (even the cocoasharp wiki is flooded with &#8220;OUTDATED&#8221; notes, but there aren&#8217;t any updates!).&nbsp; i did manage to find some that provide &#8220;better&#8221; instructions&#8230;here&#8217;s a snipped&#8230;</P>
