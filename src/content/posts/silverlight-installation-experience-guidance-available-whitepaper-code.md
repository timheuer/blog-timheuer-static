<p>I’ve been promising this for a while and I’ve ranted about creating good installation experiences for <a href="http://silverlight.net/"><strong>Silverlight</strong></a> applications before as well <a href="http://timheuer.com/blog/archive/2008/12/02/silverlight-install-experience-best-practices-netflix.aspx">here</a> and <a href="http://timheuer.com/blog/archive/2008/03/25/creating-a-great-silverlight-deployment-experience.aspx">here</a>.  Well, our team rallied and created some new guidance and sample code to provide for you to implement the best possible experience.  The guidance whitepaper can be found on the <a href="http://silverlight.net/">Silverlight Community Site</a> in the <a href="http://silverlight.net/learn/whitepapers.aspx">whitepapers</a> section and includes:</p>  <ul>   <li>Documentation for defining a good user experience</li>    <li>Sample code for media applications</li>    <li>Sample code for non-media applications</li>    <li>Sample code for implementing various states: not installed, upgrade needed, browser restart detection</li> </ul>  <p>The sample code is easy to understand and was templated so that you could change things to your brand/application.  The media sample can be dropped into your Expression Encoder templates easily and does a lot of the magic for you without having to thing (i.e., getting sizes of media, snapshot of media for image, etc.).</p>  <p>Hopefully this will help you understand the process more and provide you with the information (and code) you need to complete the best possible experience in your site.  You’ve spent the time developing your application, make sure you spend the time delivering the best install experience for those who may not have Silverlight yet.</p>  <p><a href="http://silverlight.net/learn/whitepapers.aspx">Download the whitepaper and code today</a>.</p>  <p>Hope this helps!</p>  <div style="padding-bottom: 0px;
