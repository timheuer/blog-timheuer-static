<p>One of the great things I like about some of our platform products is that they are building in extensibility more and more.  Take <a href="http://writer.live.com">Windows Live Writer</a> as an example.  It’s no secret on this blog that I’ve got a geek affair with that tool.  I use it daily and have customized it (via <a href="http://www.flickr4writer.com">plugins</a>) and my blogging platform (<a href="http://www.subtextproject.com/">Subtext</a>) to make it even more of a best experience for me for web authoring.</p>  <p>Writing plugins for Writer has been a lot of fun and a great way to get the functionality I want/need into a workflow without having a different utility to work in.  Another one of these tools has been Expression <a href="http://www.microsoft.com/expression/products/Overview.aspx?key=encoder">Encoder 2</a> which I’ve been using a bit more lately.  Expression Encoder is a tool that enables the encoding of audio/video assets into VC-1 formats and H.264/AAC formats.  It’s a really simple tool to use and also comes with several <a href="http://silverlight.net">Silverlight</a> player templates that you can choose as a part of your output.  In one click you can have your HD home movie encoded and a rich playback experience developed for you as well.  I’ve wrote several times about Encoder, templates, etc. before and you can see some of them here:</p>  <ul>   <li><a href="http://timheuer.com/blog/archive/2009/01/09/re-using-expression-encoder-templates-for-streaming-media.aspx">Using Encoder 2 templates for existing media</a></li>    <li><a href="http://timheuer.com/blog/archive/2008/01/28/automating-encoding-with-workflow-and-expression-encoder.aspx">Automating an Encoding Pipeline</a></li>    <li><a href="http://timheuer.com/blog/archive/2008/11/03/use-expression-encoder-templates-in-silverlight-application.aspx">Using Encoder templates in your Silverlight applications</a></li> </ul>  <ul>Once you are done encoding the content, you want a place to put it.  The Expression team has <a href="http://www.microsoft.com/downloads/details.aspx?FamilyID=0708e7d7-9ba1-448e-9c82-3d71e8979a1b">developed a publishing plugin</a> to have the content upload to <a href="http://silverlight.live.com">Silverlight Streaming</a> services (which is a free service).  With this type of plugin, again in one click you can have encoded, player output, and upload with ease.  This type of plugin is called a publishing plugin and it turns out is a part of the extensible platform for Encoder.  I <a href="http://timheuer.com/blog/archive/2007/12/11/write-a-publishing-plugin-for-expression-encoder-silverlight.aspx">wrote about this actually over a year ago</a> with regard to the first release of Encoder.  On <a href="http://msdn.microsoft.com/en-us/magazine/cc136758.aspx">MSDN there is a sample of an FTP server publishing point plugin</a> that <a href="http://blogs.msdn.com/webnext">Laurence Moroney</a> authored.  The <a href="http://www.microsoft.com/downloads/details.aspx?FamilyId=9A077A3D-58CE-454C-B486-153F0578BE4A&amp;displaylang=en">latest version of the SDK for Encoder 2 SP1</a> also includes another sample project (in C# and VB) of writing a publishing plugin.</ul>  <p>With no shortage of information on how to do it, I got home last night and began cranking one out.  I’ve been using <a href="http://www.amazonaws.com">Amazon’s S3 web services</a> for a while and have really grown to like it a lot.  One of the Live Writer extensions I spoke of earlier is a plugin for <a href="http://codeplex.com/s3browser">S3 for Live Writer</a> that Aaron Lerch helped out with as well!  I though I should extend Encoder so that I’d have a one click publishing point to my S3 account instead of having to use S3Fox all the time (which is an awesome tool btw).</p>  <p><img style="margin: 0px 10px 5px 0px; display: inline" align="left" src="http://s3.amazonaws.com:80/storage.timheuer.com/s2publishplugin.png" />So after getting home from a user group I started cranking one out, figuring out the nuances and just coding something together.  A few hours later I came up with what I’m calling 1.0 beta of my plugin.</p>  <p>It’s not a fancy UI, but it doesn’t need to be, it serves a purpose: <strong><em>enable publishing of Encoder output directly to an Amazon S3 bucket in one click</em></strong>.  That’s it.  Encoding just media?  No problem.  Adding a template?  Not a problem either.  You simply need to enter your Amazon S3 account information and enter a bucket.  If the bucket isn’t there, it will attempt to create it.  You can also list your current buckets if you forgot them.</p>  <p>There are likely indeed a few problems and some fit-n-finish needed.  I am positive the error handling needs to be refined as well as it could probably benefit from some more efficient threading handling.  The cool think about the Encoder publishing plugins is that they are WPF user controls, so it gave me a chance to work with more XAML.</p>  <p>At any rate, even in the current form (which isn’t perfect but seems to be working for the specific need I built it for -- “works on my machine” warranty applies here) I wanted to share it out for any others to use and hopefully give feedback and contribute.  It’s available as a <a href="http://www.opensource.org/licenses/ms-pl.html">Ms-PL</a> licensed project with source code and you can get it on CodePlex: <a href="http://www.codeplex.com/s3publishplugin"><strong>Amazon S3 Encoder Publishing Plugin</strong></a>.  I hope you like it and can give feedback.  Please if you find issues log them in the <a href="http://www.codeplex.com/s3publishplugin/WorkItem/List.aspx">Issue Tracker</a> for the project so they are trackable.</p>  <div style="padding-bottom: 0px; margin: 0px; padding-left: 0px;
