<p>for those up on the provider model&hellip;the provider toolkit went &ldquo;live&rdquo; on msdn today &mdash; parts of it have been available recently, but not as public&hellip;here&rsquo;s a central place to watch&hellip;from <a href="http://blogs.msdn.com/bgold" target="_blank">bgold</a>:</p>
<blockquote>
<p>The <a href="http://msdn.microsoft.com/asp.net/provider/" target="_blank">ASP.NET 2.0 Provider Toolkit </a>went live on MSDN today. It contains the source code for the Access database providers (which were cut from B2 and will not be in the final RTM product either, we are pushing SQL Express as the new defaults).&nbsp; Coming in mid-month will be the detailed Provider Toolkit whitepaper (probably around August 15th, it is like 75 pages in Word!!)&nbsp;that explains how developers and architects can customize and extend the provider model of ASP.NET 2.0 for any enterprise environment. This is great news as it enables developers to see sample code of the providers and gives them insight into how we built our internals.&nbsp; You can download it here: <a href="http://msdn.microsoft.com/asp.net/provider" target="_blank">http://msdn.microsoft.com/asp.net/provider</a>.</p>
<p>In Phase 2, we plan to extend the Provider Toolkit to include source code from the default SQl Server and Windows AD/AZMAN providers that will ship as part of ASP.NET 2.0.</p></blockquote>
