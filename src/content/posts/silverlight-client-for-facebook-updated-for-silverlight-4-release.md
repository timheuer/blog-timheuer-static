<p>Yet again, we’ve updated the <a href="http://www.silverlight.net/content/samples/apps/facebookclient/"><strong>Silverlight Client for Facebook</strong></a> for the <strong><a href="http://www.silverlight.net">Silverlight</a> 4</strong> release version.  In order to use the updated one, you must follow these instructions:</p>  <ul>   <li>First, <strong>uninstall the previous version you have</strong>.  This can be done in <em>Add/Remove Programs</em> on Windows or by just deleting the app on Mac. </li>    <li>Ensure you have Silverlight 4 installed.  If you are using the development tools and have installed Silverlight 4 developer tools, that’s fine.  If you are not a developer, visit <a href="http://microsoft.com/getsilverlight">http://microsoft.com/getsilverlight</a> to get the latest Silverlight 4 version (4.0.50401.0). </li>    <li><strong>After</strong> you have Silverlight 4 installed, <a href="http://www.silverlight.net/content/samples/apps/facebookclient/">visit the app page</a> and install it. </li> </ul>  <p>You should be good after this!  </p>  <h2>So why so many uninstall/re-installs!?  I thought you had an auto-update mechanism!</h2>  <p>We do!  With Silverlight 4 trusted applications, the update mechanism requires that your applications be signed in order to use the auto-update APIs.  We wanted to wait until SL4 released in order to provide a signed version (and our internal signing process didn’t allow signing the beta).  Now that we have a signed version, you’ll be prompted when an update has been installed for the app.  For more details on signing your Silverlight applications see:</p>  <ul>   <li><a href="http://timheuer.com/blog/archive/2010/03/15/whats-new-in-silverlight-4-rc-mix10.aspx#xap-signing">XAP Signing for Silverlight 4</a> </li>    <li><a href="http://www.silverlight.net/learn/videos/all/xap-signing">Video: How to sign your XAPS</a> </li> </ul>  <p>Hope this helps!
