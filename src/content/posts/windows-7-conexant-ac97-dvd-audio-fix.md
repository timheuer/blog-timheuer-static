<p>Last night I decided to mess around with some older laptops.  I had an HP Pavillion ze2000 CTO laptop (not sure why they call it “CTO” – I guess the Altec Lansing speakers included makes it an executive ;-)).</p>  <p><img style="margin: 0px; display: inline" title="Works on my machine logo" alt="Works on my machine logo" align="left" src="http://storage.timheuer.com/works-on-my-machine-starburst_3.png" />I installed Windows 7 (32-bit) from my home Windows Deployment Server with no problem.  This one took longer than the less-than-20-minute installs I’d been experience.  I’m not sure why.  It wasn’t terribly longer, but about 30-40 minutes I think.  I ran Windows Update and it got all the latest drivers.  A quick check in Device Manager showed no device problems.  Sweet.  I began testing things.  Audio playback was the last thing I played around with.  System sounds, MP3, WMV audio, CD audio…all worked.  Then I put in a DVD (Pirates of the Caribbean to be specific).  The machine suddenly sounded like a machine gun shooting back at me.  Video playback was fine, but audio playback couldn’t work.  I spent a few minutes tinkering then headed to the interweb for help.  My search was <strong><em>hp ze2000 dvd audio not working windows 7</em></strong> (I type that here in hopes to help some others).  I tried other search terms (<strong>dvd audio windows 7 ze2000</strong>, <strong>dvd audio problems windows 7</strong>) and yielded others having problems but no working solution.  Some suggested bad drivers and pointed to older drivers (which I tried and wouldn’t install).</p>  <p>Late in the night I sent a note to my trusty colleague on the Windows test team.  We send a few notes back and forth, then he looped another colleague in and asked me to send the DXDiag report.  2 seconds later I got an email with the classic “try this” note and a few steps.  Boom.  It worked.  I share the <em>works on my machine</em> solution here…it was one of those dreaded checkboxes you wouldn’t think to check.</p>  <blockquote>   <p>NOTE: This solution may work and may not.  The fact is that the laptop I have isn’t even a Vista certified machine and the Conexant AC’97 audio driver isn’t supported.  The rep I later talked to was surprised audio even worked at all with Windows 7.  It all works now though!  Your mileage may vary.</p> </blockquote>  <p>THIS IS ONLY A WORK AROUND IF YOU THINK YOU HAVE A SAME PROBLEM ON OLDER MACHINE WHICH USES THE ABOVE AUDIO DRIVER – no need to mess with things working already!</p>  <p>First start up the Sound control panel applet and you’ll see the speakers listed there.  Windows Update found the generic <em>Conexant ACLink Audio</em> driver and installed it already – it indicated that was the latest (date stamp of 2005).</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Windows 7 Sound control panel applet" alt="Windows 7 Sound control panel applet" src="http://storage.timheuer.com/conexant-fix-window1.PNG" /></p>  <p>Then choose the properties of the speakers and head to the advanced tab.  When there, UNCHECK the box that reads <em>Allow applications to take exclusive control of this device</em>.</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Speaker properties advanced tab" alt="Speaker properties advanced tab" src="http://storage.timheuer.com/conexant-fix-window2.PNG" /></p>  <p />  <p>Once unchecked, my DVD audio worked.  Other audio still worked as well.  </p>  <p>Random checkbox setting huh?  I would have taken a long time to get to that setting.  Hopefully this helps others who might be braving upgrading older laptops to Windows 7.  And yes, it runs Windows 7 great.</p>  <div style="padding-bottom: 0px; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; float: none; padding-top: 0px"
