<p>Are you using SharePoint and want to know how to leverage <a href="http://silverlight.net">Silverlight</a>?  Have you seen the <a href="http://www.codeplex.com/SL4SP">Blueprints project on Codeplex</a>?  But what if those particular implementations don’t meet your needs and you want to create your own contained web part with your Silverlight 2 application?</p>  <p>Kirk Evans writes about a method of doing this in a recent post: <strong><a href="http://blogs.msdn.com/kaevans/archive/2008/10/08/hosting-silverlight-in-a-sharepoint-webpart.aspx">Hosting Silverlight in SharePoint</a></strong>.  Take a look at Kirk’s post with a step-by-step guide of getting started with his method.  In looking at the method he describes, it is conceivable that you actually could make a more generic web part that serves as a XAP host and allows the user to set the property of the Source to a URI for a Silverlight 2 application.</p>  <p>Hmmm…I wonder if Kirk is up to modifying it ;-).</p>  <p><strong>UPDATE (08 OCT 2008):</strong></p>  <p>Looks like there is a similiar web part in the works already!  Thanks to a note from <a href="http://rialight.net/">Andy Nogueira</a>, he has a project (also on Codeplex) for <a href="http://www.codeplex.com/silverpart"><strong>SilverPart</strong></a>, which is a Silverlight wrapper web part for SharePoint.  Thanks for the pointer Andy!</p>  <div style="padding-bottom: 0px; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; float: none; padding-top: 0px" id="scid:0767317B-992E-4b12-91E0-4F059A8CECA8:a9a58e40-e47c-4385-a3ba-b1b5d1ccda4b" class="wlWriterEditableSmartContent"></div>
