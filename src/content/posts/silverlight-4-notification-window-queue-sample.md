<p>One of the cool features for <a href="http://silverlight.net">Silverlight</a> 4 Out-of-browser applications that was announced as a part of the <a href="http://timheuer.com/blog/archive/2009/11/18/whats-new-in-silverlight-4-complete-guide-new-features.aspx#toast">Silverlight 4 beta release</a> is the <strong>NotificationWindow</strong> class.  Most commonly referred to as “toast” these are the little notifications that popup in the system areas of your operating system to provide helpful (and sometimes annoying) information.  Any user of Microsoft Outlook no doubt has seen them.  They look something like this (Windows system tray area shown here):</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Silverlight 4 NotificationWindow sample image" alt="Silverlight 4 NotificationWindow sample image" src="http://storage.timheuer.com/sl4-notificationsample.png" /></p>  <p>After reading <a href="http://www.85turns.com/2009/11/18/notificationwindow-silverlight-4/">Corey’s post showing a quick FAQ</a> on customizing the experience of the NotificationWindow, I was drawn to the comment he made about queuing.  While the NotificationWindow API doesn’t have a queuing mechanism, .NET does and we can leverage that.</p>  <blockquote>   <p><strong>NOTE</strong>: For a simple overview with a short video and downloadable sample code in C# and Visual Basic about the Notification Window API visit this video: <strong><a href="http://www.silverlight.net/learn/videos/Silverlight-4-Beta-Videos/Notification-API">Notification API in Silverlight 4</a></strong>.</p> </blockquote>  <p>Let’s say you had some need to queue up notifications.  Or better yet, you really didn’t have a *need* to, but wanted to ensure that any notification you wanted to display <em>would</em> regardless of if a notification was currently showing, etc. – you want to make sure your notification hits.</p>  <p>Instead of just firing .Show() on your NotificationWindow, thing of using the <a href="http://msdn.microsoft.com/en-us/library/7977ey2c.aspx">Queue&lt;T&gt;</a> class.  This enables a collection of items stored in sequential order and serves this purpose well.  Let’s assume a simple demonstration here.</p>  <p>We have an <em>extremely</em> simple UI with four buttons…each going to trigger a different NotificationWindow.  Here’s our XAML:</p>  <div style="border-bottom: silver 1px solid; <p>In the code above <em>_currentWindow</em> is a NotificationWindow and <em>_notifyQueue</em> is a Queue&lt;NotificationWindow&gt; member variable.  We basically are adding the NotificationWindow objects we are creating to this collection and letting the Queue&lt;T&gt; help us manage what is showing when.  It is also key to note that when we do add a new NotificationWindow that we are attaching a Closed event handler that helps trigger the Dequeue function for our collection.</p>
