<p>Today (19 Oct 2009) the Visual Studio team released the second beta for Visual Studio 2010 to the public.  This is a significant milestone for the team and a huge improvement over the previous beta in my opinion as a user.  As a developer, you can find out how/when you can <strong><a href="http://go.microsoft.com/fwlink/?LinkID=151797">download Visual Studio 2010 and .NET Framework 4 beta 2 from here</a></strong>.</p>  <p>After installing the tools, one thing you may notice right away is a different look of branding of Visual Studio going forward for now.  Gone is the beloved multi-colored infinity looking thing (that’s what I call it at least) and enter the updated logo.</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Visual Studio 2010 brand logo" alt="Visual Studio 2010 brand logo" src="http://storage.timheuer.com/vs2010logo.png" /></p>  <p />  <p />  <p>I’d encourage you to download it when you can (MSDN Subscribers can do that today, general availability on Wednesday, 21 Oct) and start playing around with it.</p>  <h2>What’s new for <a href="http://silverlight.net">Silverlight</a> developers in VS2010?</h2>  <p>Well, the good news is no <a href="http://timheuer.com/blog/archive/2009/05/20/silverlight-and-visual-studio-2010-beta.aspx">more work around hacks</a> to get Visual Studio 2010 working with Silverlight development!  So what happens now when you install.  Here’s my experience from a clean machine (no existing SDKs, nor any version of Visual Studio as well).</p>  <p>After install of Visual Studio 2010 I have this for Silverlight development:</p>  <ul>   <li>Visual Web Developer </li>    <li>Silverlight 3 SDK </li>    <li>Silverlight 3 Tools (build 40818, the latest) </li> </ul>  <p>A few things missing here:</p>  <ul>   <li><a href="http://silverlight.codeplex.com">Silverlight Toolkit</a> (see below for the update to get) </li>    <li>.NET RIA Services </li> </ul>  <p>See below to get the October 2009 release of the Silverlight Toolkit to get all that goodness and support for VS2010.  Remember the installer for the toolkit also gives you the option to deploy the source (which you still have to unzip) which is EXTREMELY helpful in understanding how controls work in general as well as extending the controls to fit your own needs.</p>  <p>For .NET RIA Services, we don’t yet have a supported build for Visual Studio 2010 Beta 2.  More information on this will be coming so make sure to <a href="http://feeds.timheuer.com/timheuer">subscribe to my feed here</a> for updates and watch the forums.  I’m seeing if I can work on publishing a potential work around for RIA Services users and will post an update here if I can.  <strong><font color="#ff0000">UPDATE: </font></strong><a href="http://forums.silverlight.net/forums/p/136819/305323.aspx#305323"><strong><font color="#ff0000">View information about RIA Services roadmap and VS2010 from the team here</font></strong></a><strong><font color="#ff0000">.</font></strong></p>  <p>After installing VS2010 though, you can start developing your Silverlight applications and use the editable designer surface as well.  <a href="http://microsoft.com/expression">Expression Blend</a> will still be your friend for Visual State Manager editing and animation recording, in my opinion.</p>  <h2>Making the designer have some better performance</h2>  <p>For beta 2, there is a registry entry you can add (we did say it was beta right ;-)) to make the WPF/Silverlight designer perform better.  </p>  <blockquote>   <p><strong>NOTE: Editing your registry can be dangerous if you aren’t familiar with it.  It can cause wars, harm children and hurt your machine.  You’ve been warned.</strong></p> </blockquote>  <p>To enable this, perform these steps with all instances of Visual Studio shut down:</p>  <ol>   <li>Open regedit.exe using admin permissions (on vista/win7) </li>    <li>Navigate to HKLM\Software\Microsoft\VisualStudio\10.0 key </li>    <li>Right-click and add a new Key named “ClrHost” </li>    <li>In the new key, right-click and create a new DWORD32 with the name of StartupFlags </li>    <li>Set the value of StartupFlags to 5 </li>    <li>Close regedit and use Visual Studio as you normally would </li> </ol>  <p>I’ve also made a reg file to make this easier.  You can download this file: <a href="http://s3.amazonaws.com:80/storage.timheuer.com/Dev10DesignerFix.renametoreg">Dev10DesignerFix.renametoreg</a> and rename it to .reg and double-click it to get this entry.  I chose to force you to rename to .reg so you know what you are doing :-).  This is a step that will not be necessary in the final release version.</p>  <h2><a href="http://silverlight.codeplex.com">Silverlight Toolkit</a> October 2009 Release</h2>  <p>Additionally today, the Silverlight Toolkit published the October 2009 release of the bits.  Primarily this was for support of Visual Studio 2010 integration, but also includes drag-drop support for key controls as well as some charting and other API improvements/fixes.  You can <a href="http://silverlight.codeplex.com/Release/ProjectReleases.aspx?ReleaseId=30514">read the full details of the release of the toolkit here and download the latest build</a>.</p>  <p>Hopefully you all have a chance to start working with Visual Studio 2010.  I am looking forward to using the new IDE and features to help me be more productive!</p>  <div style="padding-bottom: 0px; margin: 0px;
