<p>Well today was the public release of <a href="http://msdn.microsoft.com/en-us/vstudio/dd582936.aspx"><strong>Visual Studio 2010 Beta 1</strong></a>.  It is the first time developers will have the chance to take it for a spin and kick the tires.  I wanted to share some information specific for <a href="http://silverlight.net/"><strong>Silverlight</strong></a> developers with regard to Visual Studio 2010 Beta 1.</p>  <p>Visual Studio 2010 is the first IDE that will support two key features for Silverlight developers: <strong>multi-targeted Silverlight development</strong> and <strong>editable design surface for Silverlight</strong>.  The second point also comes with things you might expect (like data binding wizards and dialogs as well – I’m lumping all of that in to ‘editable design surface’).  To get started there are a few things you should know.</p>  <p>First, if you are installing from a “clean” environment (perhaps a virtual machine, etc.) <em>and want the multi-target support</em>, here’s what you’ll do:</p>  <ol>   <li>Install Visual Studio 2010 Beta 1</li>    <li>Install <a href="http://www.microsoft.com/downloads/details.aspx?FamilyId=8D933343-038F-499C-986C-C3C7E87A60B3&amp;displaylang=en">Silverlight 2 SDK</a> (if you attempt to run the Silverlight 2 tools installer it will fail with an error message…just install the SDK).</li>    <li>Install <a href="http://go.microsoft.com/fwlink/?LinkID=143435">Silverlight 3 Beta SDK</a> (again, don’t attempt the tools installer as it will fail)</li>    <li>Install <a href="http://go.microsoft.com/fwlink/?LinkID=143433">Silverlight 3 Beta Developer Runtime</a></li> </ol>  <p>Once you have done this, you now have a Silverlight 2 and 3 development machine for Visual Studio 2010 Beta 1.  Congratulations!</p>  <h2>Hey, where’s my multi-targeting?!</h2>  <p>Be patient.  When you create a new project you’ll see just the standard Silverlight application templates.  Select that first and <em>then</em> you will see the multi-target option in the next dialog:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" title="Visual Studio 2010 Silverlight multi-targeting" alt="Visual Studio 2010 Silverlight multi-targeting" src="http://s3.amazonaws.com:80/storage.timheuer.com/vs2010-sl-1.png" /></p>  <p>Once you select this option you will be starting development using that runtime version.  So remember, it is <em>after</em> you select a Silverlight project type.  If you want to change the target version runtime <em>after</em> you’ve created a project, just right-click on the Silverlight project, choose the properties and you’ll see <em>Target Silverlight Version</em> and can change it there.</p>  <h2>What about the editable design surface?</h2>  <p>Once you have a Silverlight project ready, you can use the design surface for editing and dragging/dropping UI elements, arranging layout, etc.:</p>  <p><img style="display: block; float: none; margin-left: auto; margin-right: auto" src="http://s3.amazonaws.com:80/storage.timheuer.com/vs2010-sl-3.png" /></p>  <p />  <p>You can also now select items on the design surface and manipulate binding or other properties in the ‘normal’ Visual Studio way (property panes, dialogs).</p>  <h2>Hey, what happened to my Silverlight Templates…and what about .NET RIA Services?!</h2>  <p>For right now in Visual Studio 2010 Beta 1, the <em>Silverlight Navigation Application</em> template is not available as a part of the SDK installer (it’s actually a part of the tools installer…which you can’t run for VS2010 Beta 1).  If you want that template, just export one from Visual Studio 2008 and import it into this environment.  It will then show up under the <em>My Templates</em> section.</p>  <p><strong>.NET RIA Services</strong> also will not install for VS2010 Beta 1 right now.  So if you want to play around with those bits, stick to VS2008 SP1.</p>  <p>Both of these are known and will be resolved in the future.  Hopefully you understand that the products are in varying beta stages (VS2010, Silverlight 3 and RIA Services) and are not in sync right now.  Kind of a pain, but it’s the sting of early adoption I suppose.</p>  <h2>My Recommendation</h2>  <p>So what should you do?  Well I <strong><em>do</em></strong> think you should try Visual Studio 2010 and play around with it.  If you need RIA Services development, then stick with Visual Studio 2008.  If you have the ability to run a second machine or a virtual machine, I recommend putting Visual Studio 2010 in that environment.  For Silverlight 3 we will be targeting VS2008 SP1 for release.  Obviously the team is working on supporting VS2010, but for beta 1, we just couldn’t get it in time.  So I’d personally recommend sticking with VS2008 as your primary dev environment for all Silverlight 3 goodness…and run VS2010 in a separate space to play around with.  </p>  <p>Hope this helps!</p>  <div style="padding-bottom: 0px; margin: 0px;
