<p>after getting back from <a href="http://msdn.microsoft.com/events/pdc">PDC</a> last week, i can tell you that <a href="http://msdn.microsoft.com/sharepoint"><a title="SharePoint" href="http://www.microsoft.com/sharepoint" target="_blank">sharepoint</a> developers </a>implementing web parts now is a *good* thing.  after seeing <a href="http://www.asp.net/whidbey">whidbey </a>at pdc, an interesting roadmap emerged with regard to web parts, asp.net 2.0 (<a href="http://www.asp.net/whidbey">whidbey</a>) and <a href="http://msdn.microsoft.com/sharepoint">windows <a title="SharePoint" href="http://www.microsoft.com/sharepoint" target="_blank">sharepoint</a> services</a>.  web parts is a core technology in the next version of the asp.net framework...providing a “portal framework” technology.</p>
<p>good news: all web parts currently developed for wss will continue to work under this framework!</p>
<p>better news: web parts developed under whidbey will work in wss!*</p>
<p>there is a slide deck presentation from pdc that you can download...i encourage all to check it out...email me offline if you can't find it.</p>
