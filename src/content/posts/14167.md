<p>as you might suspect, if you are hosting <a href="http://silverlight.net" rel="tag">silverlight</a> applications via <a href="http://silverlight.live.com/" rel="tag">silverlight streaming</a>, you may have to make some modifications there as well to support the <a href="http://timheuer.com/blog/archive/2007/07/30/14166.aspx">release candidate</a>.  when hosting the python cube for mark, i had to make a few updates and thought i'd give some quick pointers to information that already exists as well as walk through a simple example.</p> <p>if you have a streaming sample, you might be seeing it running and showing the "Get Silverlight" banner:</p> <p><img src="http://s3.amazonaws.com/timheuer-img/stream1.png" /></p> <p>this, of course, assumes that the user has the updated silverlight plugin installed (either 1.0 RC or 1.1 alpha refresh).  this is because of two things.  first, your code might be specifically looking for a version and, perhaps more importantly, the page you have hosting the silverlight streaming application is still referencing an older silverlight.js script.  let's walk through what is needed to change my sample.</p> <p>first, i have a simple sample, it is a button that when you click does an alert().  this is a v1.0 sample, but the same steps apply for v1.1 samples.  the first step is to update my code in silverlight.  you can <a href="http://timheuer.com/blog/archive/2007/07/24/14154.aspx">view my screencast</a> on how to do this as i won't re-iterate that here.  let's assume you have your code working (updated the silverlight.js file, reviewed the breaking changes documentation to fix other things, etc.).</p> <p>okay the next thing we have to do is modify our zip file that we'll be submitting to the silverlight streaming service.  the zip file is simply a packaging mechanism that you wrap all your code up into and specify certain parameters.  there is one file in the zip file that is the manifest (appropriately named manifest.xml) that defines the parameters, and some other features.  my initial manifest file for my simple button looks like this:</p> <blockquote><pre class="csharpcode"><span class="kwrd">&lt;</span><span class="html">SilverlightApp</span><span class="kwrd">&gt;</span>
    <span class="kwrd">&lt;</span><span class="html">source</span><span class="kwrd">&gt;</span>Scene.xaml<span class="kwrd">&lt;/</span><span class="html">source</span><span class="kwrd">&gt;</span>
    <span class="kwrd">&lt;</span><span class="html">width</span><span class="kwrd">&gt;</span>400<span class="kwrd">&lt;/</span><span class="html">width</span><span class="kwrd">&gt;</span>
    <span class="kwrd">&lt;</span><span class="html">height</span><span class="kwrd">&gt;</span>50<span class="kwrd">&lt;/</span><span class="html">height</span><span class="kwrd">&gt;</span>
    <span class="kwrd">&lt;</span><span class="html">jsOrder</span><span class="kwrd">&gt;</span>
        <span class="kwrd">&lt;</span><span class="html">js</span><span class="kwrd">&gt;</span>Silverlight.js<span class="kwrd">&lt;/</span><span class="html">js</span><span class="kwrd">&gt;</span>
        <span class="kwrd">&lt;</span><span class="html">js</span><span class="kwrd">&gt;</span>Scene.xaml.js<span class="kwrd">&lt;/</span><span class="html">js</span><span class="kwrd">&gt;</span>
        <span class="kwrd">&lt;</span><span class="html">js</span><span class="kwrd">&gt;</span>Default.html.js<span class="kwrd">&lt;/</span><span class="html">js</span><span class="kwrd">&gt;</span>
    <span class="kwrd">&lt;/</span><span class="html">jsOrder</span><span class="kwrd">&gt;</span>
    <span class="kwrd">&lt;</span><span class="html">version</span><span class="kwrd">&gt;</span>0.9<span class="kwrd">&lt;/</span><span class="html">version</span><span class="kwrd">&gt;</span>
<span class="kwrd">&lt;/</span><span class="html">SilverlightApp</span><span class="kwrd">&gt;</span></pre></blockquote>
<p>note the &lt;version&gt; node.  here is where another update needs to be made.  this needs to be changed to the appropriate version you are using.  so i'll change mine to &lt;version&gt;1.0&lt;/version&gt;.  done.  for my sample, not much else actually needs to be updated in the code, so i can package up my sample and upload it.  here you'll want to login to your silverlight streaming account and choose the manage applications link on the left:</p>
<p><img src="http://s3.amazonaws.com/timheuer-img/stream2.png" /></p>
<p>once successfully updated, you'll be presented with the manage applications page again that gives you instructions on how to host your streaming application in a page (3 steps).  here's where you'll want to start making some alterations.  in step one it indicates the reference to the silverlight.js file as:</p>
<blockquote><pre class="csharpcode">&lt;script type=<span class="str">"text/javascript"</span> src=<span class="str">"http://agappdom.net/g/silverlight.js"</span>&gt;&lt;/script&gt;</pre></blockquote>
<p>for the release candidate you'll want it to be:</p>
<blockquote><pre class="csharpcode">&lt;script type=<span class="str">"text/javascript"</span> src=<span class="str">"http://agappdom.net/<strong>h</strong>/silverlight.js"</span>&gt;&lt;/script&gt;</pre></blockquote>
<p>note the subtle change in the "h" path reference.  in step 3 it indicates the code as:</p>
<blockquote><pre class="csharpcode"><span class="kwrd">function</span> CreateSilverlight(){
  Sys.Silverlight.createHostedObjectEx({</pre></blockquote>
<p>when for the release candidate it should be:</p>
<blockquote><pre class="csharpcode"><span class="kwrd">function</span> CreateSilverlight(){
  Silverlight.createHostedObjectEx({</pre></blockquote>
<p>note just removing the "Sys." as indicated in the breaking changes document in the SDKs.  making these two changes with an updated/uploaded application that works with the release candidate will make your streaming sample work again.</p>
<p>of course, these changes are documented in the news item on the silverlight streaming site posted here.  the online sdk for the silverlight streaming service will be updated shortly to reflect these changes as well.</p>
<p>so rather than host my stream in this blog post, i figure the proof is in the pudding and i updated my <a href="http://timheuer.com/blog/archive/2007/05/01/14062.aspx">previous post</a> where this exact sample was previously hosted.  <a href="http://timheuer.com/blog/archive/2007/05/01/14062.aspx">go there and you'll see a button near the bottom...click it</a> :-).</p>
<p>also, did you know you can stream your app via an iframe...use the following pattern:</p>
<blockquote>
<p><a title="http://silverlight.services.live.com/invoke/accountId/fileset/iframe.html" href="http://silverlight.services.live.com/invoke/{accountId}/{fileset}/iframe.html">http://silverlight.services.live.com/invoke/{accountId}/{fileset}/iframe.html</a></p></blockquote>
<p>pretty cool.  if you have any questions about the silverlight streaming, check out the <a href="http://silverlight.net/forums">forums</a>!  this was a very simple sample of course, but at least it should get you started in the right direction.</p>
