<p>over the past few weeks i've been getting a few requests for if my web parts work on <a title="SharePoint" href="http://msdn.microsoft.com/sharepoint" target="_blank">sharepoint</a> 2007 (also referred to as moss).  well to be honest, time has been getting the better of me and i hadn't had a chance to test.</p>
<p>well tonight i fired up a windows sharepoint services 2007 instance and loaded the web parts <strong>AS IS</strong> to see what they did</p>
<p>after that i loaded them up.  here's the flash web part in action:</p>
<p><a title="sshot-1" href="http://www.flickr.com/photos/66831573@N00/393614226/"><img alt="sshot-1" border="0" src="http://static.flickr.com/159/393614226_615895ec3e.jpg" /></a></p>
<p>and here's the <a title="" href="http://www.codeplex.com/Wiki/View.aspx?ProjectName=FeedReader">feedreader</a> in action:</p>
<p><a title="sshot-3" href="http://www.flickr.com/photos/66831573@N00/393614220/"><img alt="sshot-3" border="0" src="http://static.flickr.com/152/393614220_9920a900c6.jpg" /></a></p>
<p>they rendered without issue for me.  there is something funkadelic in the upper left corner that i didn't bother to investigate (i'm sure a weird image reference).</p>
<p>as a reminder, the <a title="" href="http://www.codeplex.com/Wiki/View.aspx?ProjectName=FeedReader">feedreader</a> does not work well at all with proxy servers requiring authentication...and the recommended configuration for proxies anyway is to use the sharepoint web.config defaultProxy settings.</p>
