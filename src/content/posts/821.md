<P>last week (early last week) i was helping a client who's machine had been infected with a hack that was adding footers in the iis configuration of a site.&nbsp; what this was doing was outputing javascript that was attempting to insert the download.ject virus on user machines that visited the site.</P>
<P>*last week* i noticed it and the companie's a/v software (mcafee) noticed it as JS/Exploit-DialogArg.B variant (not Download.Ject).&nbsp; my a/v soft (norton desktop) didn't even notice it.&nbsp; I could not find any information on the JS/Exploit-DialogArg.B virus def except on mcafee's site stating an &#8220;extremely low&#8221; warning.</P>
<P>guess what...it wasn't that &#8220;extremely low&#8221; -- as now download.ject (same thing) is all over the place...and this server *had* the appropriate suggested service pack from ms on it already.&nbsp; i admit, the clients machine looked like a different hack to get in, but i found it interesting that just now a/v corps are heightening the threat...i contacted both symantec and mcafee with the copy of my variant and they responded with &#8220;this is already been designated a low warning as microsoft already has patches that solve this&#8221; -- argh.</P>
