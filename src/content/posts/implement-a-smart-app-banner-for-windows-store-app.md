<p>As someone who uses a few iOS devices around the house, I’ve become fond of visiting sites and seeing a little banner that lets me know that a native app is available for the web app I’m using.  This concept was introduced in iOS 6 and called “<a href="http://developer.apple.com/library/ios/#documentation/AppleApplications/Reference/SafariWebContent/PromotingAppswithAppBanners/PromotingAppswithAppBanners.html">Smart App Banners</a>” in the developer documentation.  You may have seen them as well</p>  <p><img title="Smart App Banner example" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Smart App Banner example" src="http://storage2.timheuer.com/smartappbanner.png" /></p>  <p>I wanted to provide the same affordance for Windows Store apps and recalled there are already two ways to integrate the web app browsing experience with the native app in the Windows Store:</p>  <ul>   <li><a href="http://msdn.microsoft.com/en-us/library/windows/apps/Hh974767.aspx">Linking to your app</a> </li>    <li><a href="http://msdn.microsoft.com/en-us/library/ie/hh781489%28v=vs.85%29.aspx">Connect your web site to your Windows Store app</a> </li>    <li><a href="http://blogs.msdn.com/b/ie/archive/2012/06/08/high-quality-visuals-for-pinned-sites-in-windows-8.aspx">High quality visuals for Pinned Sites in Windows 8</a> </li> </ul>  <p>In iOS, this is implemented a bit natively in mobile Safari, not through any additional JavaScript files/functions.  As with any random idea I have, I always set out to search if it has been done before (often it already has).  I couldn’t find something like this specifically for Windows Store apps, however I did find an implementation for pre-iOS6 browsers as well as Android.  It was from <a href="https://twitter.com/ArnoldDaniels">Arnold Daniels</a> and hosted on GitHub, so what’s a guy to do?  Fork it!</p>  <p>And that’s what I did: <a title="https://github.com/timheuer/jquery.smartbanner" href="https://github.com/timheuer/jquery.smartbanner">https://github.com/timheuer/jquery.smartbanner</a></p>  <p>With a few more conditional checks, if you put this in your web app/site now you’ll get it automatically linked to your Windows Store app if you have the correct meta tags.  For the implementation for Windows Store apps I’ve made some assumptions about the meta tags used.  The following would get you a good experience:</p>  <div id="codeSnippetWrapper" style="overflow: auto; <p>If you have the combination of <a href="http://msdn.microsoft.com/en-us/library/ie/hh781489%28v=vs.85%29.aspx">linking your app</a> using the msApplication-ID and msApplication-PackageFamilyName values and the <a href="http://blogs.msdn.com/b/ie/archive/2012/06/08/high-quality-visuals-for-pinned-sites-in-windows-8.aspx">pinned sites high fidelity</a> msApplication-TileImage value to deliver the combined experience.  If you have all these, then the banner “just works” for you.</p>

<blockquote>
  <p><strong>NOTE</strong>: Because the Microsoft documentation mixes msApplication and msapplication (note the casing on the “A”), you should use the capitalization (which is more consistent in the documentation) for ensuring it all works smoothly.  </p>
</blockquote>

<p>When done the web app/site would display a dismissable banner (using Hulu as an example here) on a machine running Windows 8:</p>

<p><img title="Hulu smart app banner example" style="float: none; margin-left: auto; display: block; margin-right: auto" alt="Hulu smart app banner example" src="http://storage2.timheuer.com/hulusmartbanner.png" /></p>

<p>So adding one reference, 2 files and an initialization and you’ll help give your app a bit more promotional flare for those viewing in browsers other than the ‘modern’ IE view.  As you can see the screenshot above is from the desktop version which doesn’t take advantage of the other msApplication-** attribute usage in desktop mode.  The one thing it doesn’t do that the iOS system-level banner does is detect if the app is installed and change “view” to “open” however there isn’t a mechanism to easily detect if the app is installed using JavaScript from a web app.</p>

<blockquote>
  <p><strong>NOTE</strong>: While the ‘View in Store’ button works in IE, Safari and Firefox it does not work in Chrome.  I’m not sure what any timeframe it would be for Chrome to support the store launch protocol.</p>
</blockquote>

<p>Anyhow, I thought this might be a useful addition to app developers web apps that have a Windows Store app counterpart (and perhaps already have an iOS and Androi counterpart as well).  I’ve submitted a pull request to the project I forked from, but you can get it from my <a href="https://github.com/timheuer/jquery.smartbanner">GitHub repository for jquery.smartbanner</a>.</p>
