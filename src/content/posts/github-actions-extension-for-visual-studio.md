<p>I LOVE GitHub Actions! I’ve written about this a lot and how I’ve ‘seen the light’ with regard to ensuring CI/CD is a part of any/every project from the start. That said I’m also a HUGE Visual Studio fan/user.&#160; I like having everything as much as possible at my fingertips in my IDE and for most basic things not have to leave VS to do those things.&#160; Because of this I’ve created <a href="https://marketplace.visualstudio.com/items?itemName=TimHeuer.GitHubActionsVS"><strong>GitHub Actions for Visual Studio</strong></a> extension that installs right into Visual Studio 2022 and gives you immediate insight into your GitHub Actions environment.&#160; </p>  <p><img title="Screenshot of light/dark mode of extension" style="margin: 0px auto; float: none; display: block; background-image: none;" border="0" alt="Screenshot of light/dark mode of extension" src="https://storage2.timheuer.com/ghforvs1.png" width="1337" height="890" /></p>  <blockquote>   <p>Like nearly every one of my projects it starts as completely selfish reasons and tailored to my needs. I spend time doing this in some reserved learning time and the occasional time where my family isn’t around and/or it’s raining and I can’t be on my bike LOL. That said, it may not meet your needs, and that’s okay.</p> </blockquote>  <p>With that said, let me introduce you to this extension…</p>  <h2>How to launch it</h2>  <p>First you’ll need to have a project/solution open that is attached to GitHub.com and you have the necessary permissions to view this information. The extension looks for GitHub credentials to use interacting with your Windows Credential manager. From VS Solution Explorer, right-click on a project or solution and navigate to the “GitHub Actions” menu item.&#160; This will open a new tool window and start querying the repository and actions for more information.&#160; There is a progress indicator that will show when activity is happening.&#160; Once complete you’ll have a new tool window you can dock anywhere and it will show a few things for you, let’s take a look at what those are.</p>  <h3>Categories</h3>  <p>In the tool window there are 4 primary areas to be aware of:</p>  <p><img title="Screenshot of the tool window annotated with 4 numbers" style="margin: 0px auto; float: none; display: block; background-image: none;" border="0" alt="Screenshot of the tool window annotated with 4 numbers" src="https://storage2.timheuer.com/ghforvs2.png" width="536" height="837" /></p>    <p>First in the area marked #1 is a small toolbar.&#160; The toolbar has two buttons, one to refresh the data should you need to manually do so for any reason. The second is a shortcut to the repository’s Actions section on GitHub.com.</p>  <p>Next the #2 area is a tree view of the current branch you have open and workflow runs that targeted that.&#160; It will first show executed (or in-progress) workflow runs, and then you can expand it to see the jobs and steps of each job.&#160; At the ‘leaf’ node of the step you can double-click (or right-click for a menu) and it will open the log for that step on GitHub.com directly.</p>  <p>The #3 area is a list of the Workflows in your repository by named definition. This is helpful just to see a list of them, but also you can right-click on them and “run” a workflow which triggers a dispatch call to that workflow to execute!</p>  <p>Finally the #4 area is your Environments and Secrets. Right now Environments just shows you a list of any you have, but not much else. Secrets are limited to Repository Secrets only right now and show you a list and when the secret was last updated.&#160; You can right-click on the Secrets node to add another or double-click on an existing one to edit.&#160; This will launch a quick modal dialog window to capture the secret name/value and upon saving, write to your repository and refresh this list.</p>  <h3>Options</h3>  <p>There are a small set of options you can configure for the extension:</p>  <p><img title="Screenshot of extension options" style="margin: 0px auto; float: none; display: block; background-image: none;" border="0" alt="Screenshot of extension options" src="https://storage2.timheuer.com/ghforvs3.png" width="890" height="387" /></p>  <p>The following can be set:</p>  <ul>   <li>Max Runs (Default=10): this is the maximum number of Workflow Runs to retrieve</li>    <li>Refresh Active Jobs (Default=False): if True, this will refresh the Workflow Runs list when any job is known to be in-progress</li>    <li>Refresh Interval (Default=5): This is a number in seconds you want to poll for an update on in-progress jobs.</li> </ul>  <h2>Managing Workflows</h2>  <p>Aside from viewing the list there are a few other things you can do using the extension:</p>  <ul>   <li>Hover over the Run to see details of the final conclusion state, how it was triggered, the total time for the run, and what GitHub user triggered the run</li>    <li>If a run is in-progress, right-click on the run and you can choose to Cancel, which will attempt to send a cancellation to stop the run at whatever step it is in</li>    <li>On the steps nodes you can double-click or right-click and choose to view logs.&#160; This will launch your default browser to the location of the step log for that item</li>    <li>From the Workflows list, you can right-click on a name of a Workflow and choose “Run Workflow” which will attempt to signal to run the start a run for that Workflow</li> </ul>  <h2>Managing Secrets</h2>  <p>Secrets right now are limited to Repository Secrets only.&#160; This is due to a limitation the Octokit library this extension uses.&#160; If you are using Environment Secrets you will not be able to manage them from here.&#160; </p>  <p><img title="Screenshot of modal dialog for secret editing" style="margin: 0px auto; float: none; display: block; background-image: none;" border="0" alt="Screenshot of modal dialog for secret editing" src="https://storage2.timheuer.com/ghforvs4.png" width="734" height="323" /></p>  <p>Otherwise:</p>  <ul>   <li>From the Repository Secrets node you can right-click and Add Secret which will launch a modal dialog to supply a name/value for a new secret. Clicking save will persist this to your repo and refresh the list.</li>    <li>From an existing secret you can double-click it or right-click and choose ‘Edit’ and will launch the same modal dialog but just enables you to edit the value only.</li>    <li>To delete a secret, right-click and choose delete. This is irreversible, so be sure you want to delete!</li> </ul>  <h2>Get Started and Log Issues</h2>  <p>To get started, you simply can navigate to the <a href="https://marketplace.visualstudio.com/items?itemName=TimHeuer.GitHubActionsVS">link on the marketplace</a> and click install or use the Extension Manager in Visual Studio and search for “GitHub Actions” and install it.&#160; If you find any issues, the source is available on my GitHub at <a href="https://github.com/timheuer/GitHubActionsVS"><strong>timheuer/GitHubActionsVS</strong></a> but also would appreciate that you could log an Issue if you find it not working for you.&#160; Thanks for trying it out and I hope it is helpful for you as it is for me.</p>
