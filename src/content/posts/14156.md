<p>i'm sitting in <a href="http://andigutmans.blogspot.com/" rel="tag">andi gutman</a>'s presentation at <a href="http://conferences.oreillynet.com/os2007/" rel="tag">oscon</a> about building rich internet applications in <a href="http://www.php.net">php</a>.  my expectations will be that this will be talking about ajax and flex (given the nature of the conference).</p>  <p>andi's first talking about the nature of ajax for rich applications in php.  what is cool is that he mentioned the <a href="http://ajax.asp.net">micorosft ajax client library</a> and the <a href="http://www.codeplex.com/phpmsajax">codeplex php kit</a> in the talk so far.  he said he likes talking about it because it demonstrates the nature of ajax standards.  </p>  <p>he also talked initially about <a href="http://www.w3schools.com/php/php_xml_simplexml.asp">simplexml</a> as a mechanism for communicating with ajax services.  i used simplexml myself in the learn2asp.net series for php that i helped out with.  i found the microsoft ajax implementation easier to consume even if there might be one more task of implementing the interface.</p>  <p>andi's now showing a chat application build in ajax/php.  it's a basic chat application, but then integrates yahoo keywords services, highlighting keywords in the chat text.  then clicking on the text integrates with flickr to get pictures based on that keyword -- providing an end-to-end seamless service.  </p>  <p>so how was the sample built?  <a href="http://www.zendframework.com/">zend framework</a>.  andi walks through the zend framework architecture, highlighting areas of json serialization, etc.  he's talking now about using the model-view-controller (MVC) pattern for developing the chat application and how the zend framework uses this (i.e., routing is one simple example, mapping URI requests to actions).</p>  <p>hmm...this really is more of an MVC using zend framework talk so far...interesting, but not what i expected so far.</p>  <p>one cool function for ajax developers: Zend_Json::encode($yourArray), oooh, even better: Zend_Json::fromXML($response-&gt;getBody())</p>  <p>so what will be in zend framework vNext? </p>  <ul>   <ul>     <li>much faster release cycles (monthly to bi-monthly mini releases; 4 times a year minor releases)</li>      <li>ajax-enabled form components in v1.1</li>      <li>ajax support in eclipse-based dev tools (javascript editing/debugging, syntax highlighting, toolkit support, code completion)</li>      <li>zend component model (server php component architecture; ajax client toolkit, client-side messaging; tooling in dev tools/ides for components)</li>   </ul> </ul>  <p>andi's giving a security talk regarding php next...think i'm headed to somewhere else though.  zend framework looks very useful for php developers.  i'd love to start seeing the mvc implementation of the zend framework run through the <a href="http://www.iis.net/default.aspx?tabid=1000051">fastcgi implementation for IIS</a> and see what php developers think of that!</p>  <div class="wlWriterSmartContent"
