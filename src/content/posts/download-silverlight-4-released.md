<p>Today’s the day!  Tuesday at DevConnections in Las Vegas, <a href="http://weblogs.asp.net/scottgu">Scott Guthrie</a> just announced the ‘launch’ of <a href="http://www.silverlight.net"><strong>Silverlight</strong></a><strong> 4</strong>.  We wanted to take the opportunity at DevConnections to let a large audience of our customers online/offline know that we’re done and shipped Silverlight 4.  As of today it’s now available for you to download/use.  Here’s some helpful quick update information for you:</p>  <ul>   <li><a href="#get-started">Download the bits</a> </li>    <li><a href="#rc">Understanding what is in the bits</a> </li>    <li><a href="#riaservices">RIA Services</a> </li>    <li><a href="#wp7dev">Windows Phone 7 Developer Tools</a> </li>    <li><a href="#vs">VS2008 and VS2010 on same machine</a> </li>    <li><a href="#other">Other resources</a> </li> </ul>  <h2>What’s New in Silverlight 4?</h2>  <p>Rather than cut/paste what I’ve already said here, I recommend the following reading:</p>  <ul>   <li><a href="http://timheuer.com/blog/archive/2009/11/18/whats-new-in-silverlight-4-complete-guide-new-features.aspx">A Complete Guide to Silverlight 4</a> </li>    <li><a href="http://timheuer.com/blog/archive/2010/03/15/whats-new-in-silverlight-4-rc-mix10.aspx">What’s new in Silverlight 4 RC</a> </li> </ul>  <p>These two posts are full of detailed information with links to tutorials and videos to help you get started.  Since the RC (which was released middle March), not much has changed from an API perspective so you should be good to go getting started using the above information.  One thing that has changed in the tools is that for XAP signing, there is a UI support for enabling the tool to help you select a certificate, etc.  You can still use my post-build method though.</p>  <h2><a name="get-started"></a>Downloading Silverlight 4</h2>  <p>The availability of Silverlight 4 will be approximately 10:00 AM PST on Thursday 15 April 2010.  I suggest you prep now by getting the Visual Studio 2010 release that was launched/available yesterday.</p>  <p>The goods:</p>  <ul>   <li><strong><a href="http://www.microsoft.com/visualstudio/">Visual Studio 2010</a></strong> or <a href="http://www.microsoft.com/express/Web/"><strong>Visual Web Developer 2010 Express</strong></a> </li>    <li><strong><a href="http://download.microsoft.com/download/5/B/4/5B46BF8A-9350-49D5-B1E7-7789818FBF41/Silverlight4_Tools.exe">Silverlight 4 Tools RC for Visual Studio 2010</a></strong> </li>    <li><a href="http://www.microsoft.com/downloads/details.aspx?FamilyID=88484825-1b3c-4e8c-8b14-b05d025e1541">Expression Blend 4 RC</a> </li>    <li><a href="http://silverlight.codeplex.com">Silverlight 4 Toolkit April 2010 Release</a> </li> </ul>  <p>That’s my own personal minimum list.  VS and SL4 tools are the minimum required to get started doing development.</p>  <h2><a name="rc"></a>Wait, what’s this RC stuff?</h2>  <p>To be clear, Silverlight 4 has released.  This is RTW (release to web).  It is the version 4 of Silverlight.  Shipped.  Done.  Finished.</p>  <p>The tools (namely SL4 tools, RIA Services and Blend) are in their ‘release candidate’ mode.  I’ll spare you the gory details, but remember that these tools teams need SL4 to be *done* before they can be done.  SL4 is a dependency for them.  These tools are release quality though and I’d recommend using them.  Their final versions will come soon enough and will be a minor update.</p>  <h2><a name="riaservices"></a>What about RIA Services?!  You don’t consider that to be valuable?!</h2>  <p>Actually I do.  But if you’ve installed the Silverlight 4 Tools, then you already have it!</p>  <p>This seems to be some confusion to many and perhaps because of how we present the information in an effort to be complete.  If you are a developer, install Silverlight 4 Tools.  After this installation completes you will have:</p>  <ul>   <li>Silverlight 4 developer runtime </li>    <li>Silverlight 4 SDK </li>    <li>Visual Studio patch, debug tools and project templates </li>    <li>WCF RIA Services RC </li> </ul>  <p>installed.  No need to run the SDK or RIA Services installers separately.  Adding the Silverlight Toolkit to this provides you with more controls to leverage in your applications as well (charting, <a href="http://timheuer.com/blog/archive/2010/03/15/whats-new-in-silverlight-4-rc-mix10.aspx#context-control">ContextMenu</a>, etc.).</p>  <h2><a name="wp7dev"></a>Hey, what about the Windows Phone 7 developer tools?</h2>  <p><strong>If you need to continue doing Windows Phone 7 development, stick with the Visual Studio 2010 Release Candidate for now!</strong>  The <a href="http://forums.silverlight.net/forums/t/175181.aspx">updated CTP of the Windows Phone developer tools is not quite done yet</a>.  Information about updated tools availability will be forthcoming on these tools.  Stay tuned.</p>  <h2><a name="vs"></a>Can I keep VS2008 and VS2010 on the same machine?</h2>  <p>Yes you can.  Visual Studio 2008 and 2010 can co-exist on the same machine.  Obviously there are some differences to what the tools can do with regard to Silverlight.  There are two distinct differences I like to call out:</p>  <ul>   <li>VS2008 <strong>cannot</strong> be used for Silverlight 4 development. </li>    <li>You can only have <strong>one</strong> version of WCF RIA Services installed on the machine. </li> </ul>  <p>For the latter, you may be asking <em>“huh?”</em>  There is a version of RIA Services for VS2008.  There is also a version for VS2010.  Unfortunately RIA Services <strong>cannot</strong> co-exist in two versions.  You have to pick one.  And the VS2008 one is only for SL3 and only supported until December this year.  I recommend moving on from that…it’s not going to be developed any more…essentially the PDC09 version was the last revision there.</p>
