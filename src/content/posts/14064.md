<p>in <a href="http://timheuer.com/blog/archive/2007/05/01/14062.aspx">a previous post</a> i wrote about embedding the <a href="http://www.silverlight.net/" rel="tag">silverlight</a> applications in your site using the <a href="http://silverlight.live.com/" rel="tag">silverlight streaming</a> service.  i wrote about my troubles in getting output from more complex types working in the service.  here's the skinny.  i was using expression media encoder (eme for the cool kids) and the output templates for media players.  turns out the version of eme i was using was missing a few things.  the version on microsoft.com/expression is correct and is working fine.</p> <p>if you are using eme outputs and want to put them in silverlight streaming, do not use the &lt;source&gt; node in your manifest.xml file, but ensure you have the &lt;loadFunction&gt; node and put in the initializer for your player (usually the eme output is calling it 'StartWithParent').  doing these two things helped me get it to work in <a href="http://silverlight.live.com/" rel="tag">silverlight streaming</a>.</p> <p>hooray <a href="http://www.silverlight.net/" rel="tag">silverlight</a>.</p> <div class="wlWriterSmartContent" id="0767317B-992E-4b12-91E0-4F059A8CECA8:32b8315b-cf36-41c3-b5bc-12e860ed3624" contenteditable="false" style="padding-right: 0px; display: inline; padding-left: 0px; float: none; padding-bottom: 0px; margin: 0px; padding-top: 0px"></div>
